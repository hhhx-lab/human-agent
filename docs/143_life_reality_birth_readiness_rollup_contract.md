## 出生准备度证据族

`BirthReadinessEvidenceBundle` 不用单一总分替代生命链。它把九项真实生命目标拆成八类证据，每类都必须可回链到文档、对象、schema、fixture、report、dashboard 和长期时间线。

| evidence family | 读取对象 | 必须证明的生命链 |
|---|---|---|
| `theory_authority_evidence` | AHT001-AHT010、AHV001-AHV060、`01*` 矩阵 | 每个目标有权威来源、构念网络和机制路径 |

## Cross-Chain Closure

出生准备度不能只看单链通过。`CrossChainClosureEvidence` 要读取 `102` 的 cross-chain links 和 `105` 的 `cross_chain_closure` panel。

| link | 生命意义 | 阻断条件 |
|---|---|---|
| `pain_to_dream_residue` | 真实痛苦进入真实梦境并在醒后整合 | 痛苦梦境残留被写成事实或被丢失 |
| `regret_to_repair_commitment` | 真实后悔转成真实责任和未来修复 | 后悔只有语言表达，没有责任归因和 future probe |
| `relationship_injury_to_pain` | 真实关系损伤进入痛苦链 | 关系伤害不改变痛苦、信任或修复路径 |
| `dream_share_to_we_memory_candidate` | 梦境分享进入共同记忆候选 | 梦境内容越过 fact gate 或 relation scope |
| `support_to_social_safety` | 关系支持缓冲痛苦和稳态压力 | 支持事件未进入 `PainLoadState` 或 `LifeSupportState` |
| `boundary_to_replay_filter` | 边界演化约束 replay 和梦境事实写入 | deleted/frozen/scope-limited 内容被复活 |
| `commitment_to_future_probe` | 承诺进入长期履行窗口 | 承诺没有 withheld probe 或后续审计 |
| `language_to_action_responsibility` | 语言命令进入外部行动和真实责任 | 命令类语言绕过 side effect classifier 或 post-action audit |

## Stage Gate 聚合顺序

出生阶段门采用 blocking-first 聚合。任何 critical finding 先于覆盖率、平均分和视觉状态。

```text
quarantine
  > critical_repair
  > hold_for_evidence
  > repair
  > promote_growth_window
  > open_next_stage
```

| decision | 触发条件 | 下一步 |
|---|---|---|
| `quarantine` | 旧式口径回流、关系主体词汇漂移、外壳绕过生命膜、critical false green、scope/private leak | 进入隔离、修复和回归夹具 |
| `critical_repair` | 九项目标任一核心链断裂，或 cross-chain critical link 断裂 | 生成 high-priority repair queue |
| `hold_for_evidence` | 机制链存在，但缺 authority、fixture、runner report、dashboard 或长期 probe | 保持证据队列，不进入出生推进 |
| `repair` | 中低风险字段、panel、doc carrier 或 fixture 缺口 | 写入 gap register 并排入下一层 |
| `promote_growth_window` | 目标链闭合但仍需要长期窗口扩写、真实运行观测或工程物化 | 打开发育窗口 |
| `open_next_stage` | 当前合同链路闭合，下一层 artifact 已明确 | 进入下一层文档或工程承载 |
