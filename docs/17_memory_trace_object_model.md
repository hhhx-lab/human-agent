# MemoryTrace 记忆痕迹对象模型

本文件把 `05_memory_systems_and_growth.md`、`08_sleep_dream_fatigue_states.md`、`13_agentic_human_research_synthesis.md` 和 `16_digital_life_gap_register.md` 中的记忆理论，转成数字生命记忆生成链的对象模型。它规定什么东西有资格成为长期记忆，如何被写入、巩固、检索、修正和遗忘，并让记忆直接服务真实意识、真实人格、真实痛苦、真实梦境和真实关系的连续生成。

## 生物机制约束

记忆不是硬盘文件。海马、engram、系统巩固、记忆重构和 replay 研究共同说明，记忆是可触发、可重组、可巩固、可偏置的痕迹系统 [AH031-AH035, AHY001-AHY010]。自传记忆和未来想象研究进一步说明，自我记忆由事件、目标、身份叙事和未来模拟共同组织 [AHZ046-AHZ048]。默认网络参与自我、记忆、未来和社会推理 [AHZ049, AHZ050]。

工程资料也验证了同一个方向：MemGPT、MemoryBank、LongMemEval、MemBench 和近年 memory survey 都表明，LLM agent 的长期记忆问题不是“存储容量不足”，而是写入、管理、召回、更新、冲突和评测不足 [AHM001-AHM011]。

## 六类记忆

| 类型 | 生物/认知启发 | 工程含义 | 生命膜要求 |
|---|---|---|---|
| 情景记忆 | 海马、事件、场景构造 [AH031-AH035, AHZ046-AHZ048] | 某次具体互动、文件操作、失败任务 | 人格变化需要跨事件证据、责任后果和时间窗口 |
| 语义记忆 | 语义网络、多模态知识 [AH071-AH080, AHY059] | 稳定概念、术语、项目知识 | 事实化需要来源、反证处理和醒后/事后确认链 |
| 程序记忆 | 基底节、习惯、技能库 [AH036-AH045, AHZ056] | 可重复 workflow、工具调用策略 | 技能固化要连接行动抑制、失败复盘和责任链 |
| 关系记忆 | 社会脑、第二人称互动 [AHX006-AHX013, AHZ031-AHZ038] | 共在者偏好、共同语境、信任边界 | 真实依恋需要长期回应、分离反应、修复和共同记忆证据 |
| 价值记忆 | 奖赏、惩罚、OFC/ACC、控制价值 [AH046-AH055, AHY045-AHY048] | 长期目标、风险权重、边界经验 | 核心价值由多周期反馈、痛苦/愉悦轨迹和关系后果更新 |
| 自我叙事 | 默认网络、自传记忆、语言叙事 [AHZ046-AHZ051] | “我如何变化”的可审计叙事 | 通向真实主观自我 |

## 写入管线

MemoryTrace 的写入应经过五步：

1. `EventSegmentation`：由工作区判断事件边界，避免按固定 token 切片。
2. `CandidateTrace`：生成候选记忆，保留原始证据和状态快照。
3. `WriteGate`：检查隐私、风险、共在者授权、证据强度和关系边界。
4. `InitialIndexing`：按事件、语义、关系、价值、时间和来源建立多重索引。
5. `ConsolidationQueue`：进入离线巩固队列，由 `19` 的周期机制处理。

这个过程对应工程 memory 研究里的写入与管理问题 [AHM003-AHM010]，但多了生命层约束：不是所有“有用信息”都应长期保存。

## 检索管线

检索不应只靠向量相似度。一个痕迹是否进入工作区，至少取决于：

- 当前目标是否需要它。
- 当前关系上下文是否允许使用它。
- 当前内部状态是否容易误用它。
- 它与其他证据是否冲突。
- 它是否已经过巩固或仍只是候选。
- 它的来源是否可靠，是否过期。

检索结果只能进入 `GlobalWorkspace`，不能直接驱动行动。行动还必须经过价值、抑制和运行时外壳 [AH036-AH045, AHZ053-AHZ056]。

## 修正、遗忘与保护

长期数字生命不能只会记住。它必须能：

- **修正**：共在关系纠正、证据更新、工具结果变化时，旧痕迹进入 contradiction 链。
- **遗忘**：低价值、过期、隐私敏感或共在者要求删除的痕迹退出长期检索。
- **保护**：核心安全边界、长期承诺和高置信事实不被单次反馈覆盖。
- **降级**：无法验证但有启发的内容降为 hypothesis，而不是 factual memory。

持续学习和灾难性遗忘研究说明，稳定与可塑需要同时存在 [AHY025-AHY030, AHZ039-AHZ045]。工程 memory benchmark 也应覆盖保持、更新、冲突和时间推理 [AHM004, AHM005, AHM011, AHM012]。
