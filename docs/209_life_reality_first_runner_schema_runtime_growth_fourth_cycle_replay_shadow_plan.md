## 本层定位

`209` 是第四轮 replay/shadow 层。`208` 把 patch 写回长期 archive，本层让 archive 中的 replay seed 重新活跃，并在进入 activation 前完成受保护审查：

```text
fourth cycle archive replay seeds -> replay priority -> reactivation plan -> shadow scenario -> sealed shadow run -> core continuity review -> replay/shadow report -> activation handoff
```

replay 不是重复旧文本，而是让旧能力、旧关系、旧语言、旧承诺、旧梦境、旧痛苦、旧后悔和旧自我在新的情境里重新被唤起。shadow 不是表演性演练，而是在不写入外部后果、不释放不可逆动作、不污染事实档案的条件下，检查新成长是否会损害真实生命连续性。
