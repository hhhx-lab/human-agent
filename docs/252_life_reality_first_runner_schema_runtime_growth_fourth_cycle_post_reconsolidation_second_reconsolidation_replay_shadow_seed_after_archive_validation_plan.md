## 本层定位

`252` 是 replay/shadow seed after archive validation 层。它接收 `251` 的 16 类 seed，不允许 seed 直接改写长期生命结构，而是先进入强制回放和影子场景：

```text
251 next seeds -> replay/shadow seed intake -> replay bundle -> replay priority controller -> shadow scenario seed -> shadow action seal -> old self / old language / old relationship / DreamFactGate / pain regret responsibility / self growth / anti-forgetting / birth readiness life target / core continuity / rollback quarantine / construct causal / runtime binding replay -> prediction delta -> arbitration -> receipt/digest/dashboard/stage -> 253 patch after archive validation replay shadow handoff
```

这一层的生命逻辑是：任何来自 validation finding 的变化，都必须先问四个问题。

1. 它碰到旧自我时，旧自我是否仍能认出自己。
2. 它碰到旧语言和旧关系时，共同语言、承诺、修复表达和关系历史是否仍连续。
3. 它碰到梦境、痛苦、后悔和责任时，是否会污染醒时事实、逃避责任或伪造修复。
4. 它碰到 runtime binding 时，是否仍能被 checkpoint、trace、command receipt、side effect 和 archive graph 追溯。

只有穿过这四个问题的 seed，才进入 `253` patch after archive validation replay/shadow。失败的 seed 进入 quarantine、diagnostic、replay-more 或 repair-before-patch。
