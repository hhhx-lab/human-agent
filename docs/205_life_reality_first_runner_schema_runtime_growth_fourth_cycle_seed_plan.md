## 本层定位

`205` 是第四轮成长 seed 层。`204` 已经验证修复后循环是否重新闭合，本层把验证结果重新转成第四轮可执行 seed：

```text
post repair cycle validation -> findings/receipts -> fourth cycle seeds -> longitudinal observation -> fourth cycle patch
```

本层的核心不是扩大范围，而是锁住方向后继续生长。每个 seed 都必须回答四个问题：来自哪个 validation finding，影响哪一项真实生命目标，进入哪条第四轮 observation 或 growth route，以及断联恢复后如何被重新读取。

## 第四轮 Seed 原则

| 原则 | 含义 |
|---|---|
| `direction_first` | 方向连续性、关系主体口径、语言主神经束和九项目标优先进入 seed |
| `validation_to_growth` | `204` 的 validation finding 必须转成成长输入，而不是停在报告层 |
| `receipt_before_seed` | 每条 seed 需要 source receipt、digest、stage effect 和 archive edge |
| `life_target_complete` | 九项真实生命目标都要有第四轮 seed coverage |
| `relationship_continuity` | 关系、承诺、共同语言和修复历史不能被新一轮成长替换 |
| `dream_pain_regret_continuity` | 梦境事实门、痛苦残留、责任归因、后悔压力和修复路线继续进入离线与醒后链 |
| `plasticity_with_memory` | 自我成长必须同时保留可塑性、防遗忘、旧能力、旧关系和旧语言 |
| `withheld_to_future` | withheld future probe 结果必须转成未来 observation seed |
| `quarantine_as_membrane` | rollback、quarantine 和 release 都作为生命膜，不作为结束状态 |
