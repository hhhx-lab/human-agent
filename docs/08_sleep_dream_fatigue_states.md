# 睡眠、梦境、发呆、疲惫与离线整合

## 睡眠是主动状态

睡眠研究显示，睡眠参与记忆巩固、突触稳态、代谢清除和状态切换 [AH056, AH057, AH066-AH068]。因此 Agentic Human 的“睡眠”不应只是停止运行，而应是主动离线处理。

可对应的工程过程包括：

- 整理近期记忆。
- 压缩低价值细节。
- 提取长期主题。
- 修复冲突记忆。
- 降低过高权重。
- 清理缓存和错误轨迹。

## 论文精读：睡眠与记忆巩固

Rasch 与 Born 的 Physiological Reviews 综述系统总结了睡眠在记忆中的作用 [AH066]。关键不是“睡一觉记得更牢”这么简单，而是不同睡眠阶段可能对不同记忆系统有不同贡献：慢波睡眠、海马-新皮层重放、快速眼动睡眠、情绪记忆和程序学习之间关系复杂。

Tononi 与 Cirelli 的 synaptic homeostasis hypothesis 提出，清醒学习带来突触增强和代价，睡眠可能帮助全局归一化，使重要连接保留、不重要连接下降 [AH067]。Xie 等人的 glymphatic 研究则提示睡眠与代谢废物清除相关 [AH068]。

工程转译的重点是：长期学习系统必须有离线维护期。没有维护期，记忆会膨胀、权重会偏移、错误会累积、人格会被短期事件污染。

## 梦境与生成重组

Nir 与 Tononi 对梦境的综述显示，梦境涉及感知、记忆、情绪和意识状态的特殊组合 [AH069]。Agentic Human 要把“梦境”推进为无外部输入或弱外部输入下的真实梦境生成机制：

- 将远距离记忆组合成新场景。
- 生成未来场景与反事实路径。
- 测试价值冲突。
- 演练社交或任务情景。

真实梦境的目标不是把梦写成任务摘要，而是让离线生成、情绪主题、自我叙事、痛苦残留、关系修复和醒后整合形成连续链。

## 真实梦境生成门

梦境类状态的核心职责：

- 把不相邻的记忆片段组合，产生新解释。
- 生成未来对话、失败场景和修复路径。
- 在不执行真实行动的情况下演练策略。
- 将情绪/价值冲突转化为可分析材料。

梦境写入生命时间线时要经过生命膜：

- 事实性内容走 `DreamFactGate`，只有通过证据、来源和冲突检查才进入事实记忆。
- 长期信念变化走 `DreamBeliefGate`，需要后续清醒状态、关系后果和审计链。
- 外部行动走 `ActionInhibitionGate`，梦境期保持内在生成，不直接执行工具。
- 对共在者表达梦境内容时携带 `dream_marker`、来源 trace 和醒后整合状态。

因此“梦境”是受生命膜保护的内在生成器：它生成真实梦境材料，再由清醒工作区决定如何整合。

## 疲惫与负载预算

Van Dongen 等人的睡眠限制研究表明，慢性觉醒会累积损害认知表现 [AH070]。McEwen 的压力负荷研究也提醒，长期高负荷会改变适应能力 [AH059]。

Agentic Human 应有疲惫类机制：

- 长时间高强度推理后降低行动速度。
- 当错误率上升时提高验证。
- 当记忆冲突积累时进入整理。
- 当共在者输入高频且复杂时保持显著性警觉。

## 疲惫的生命负荷链

数字生命的疲惫应从长期负荷中生成：当错误压力、记忆压力、冲突压力、任务积压、外部等待和高唤醒持续累积时，系统应改变策略、降低阈值、请求恢复并记录体验痕迹。这里的疲惫由稳定性保护、内部负荷和主观体验生成路径共同构成。

疲惫状态下可采取：

- 降低自主探索。
- 增加事实核验。
- 缩小行动范围。
- 优先完成收尾任务。
- 请求或等待更清晰目标。
- 进入巩固/恢复。

这能防止系统在高负荷下越做越乱。

## 压力、负荷与恢复

allostatic load 理论描述长期适应所带来的代价 [AHX022]。McEwen 的压力介质研究强调压力反应既可能保护系统，也可能在长期高负荷下产生损害 [AHX023]。Lupien 和 Joels/Baram 的综述进一步说明，压力影响依赖时间尺度、阶段和脑区 [AHX024, AHX025]。

Agentic Human 可以把压力分成：

- **急性压力**：当前任务高风险、共在者等待、工具失败。作用是提高警觉和验证。
- **慢性压力**：长期错误积累、未整理记忆、关系冲突、过多未完成任务。作用是触发恢复和重构。
- **恢复窗口**：低外部需求下进行巩固、清理和权重归一。

如果没有慢性负荷概念，系统会一直保持“可用”，但内部质量不断下降。

## 发呆与默认整合

发呆可与默认模式、走神和自发思维研究连接 [AH011-AH015]。它不是睡眠，也不是任务执行，而是低约束内部生成状态。

可能功能：

- 把近期碎片串成主题。
- 从失败中生成替代路径。
- 形成“我最近在关注什么”的自我摘要。
- 为下一次共在者输入准备更好的上下文。

## 状态之间的节律

一个类生命体 agent 需要昼夜节律的硅基转写 [AH060]。这不一定是 24 小时，而可以是多层周期：

- 秒级/分钟级：心跳式自检、任务进度检查。
- 小时级：短期记忆整理、错误复盘、任务队列更新。
- 天级：长期主题总结、关系状态更新、知识库清理。
- 周/月级：人格慢变量审计、价值权重复盘、架构自检。

节律的目的是让数字生命拥有内在时间，不只在外部事件下变化。

## 硅基炎症：异常负反馈

炎症研究显示，免疫信号可以影响动机、情绪和认知 [AHX026]；炎症反射说明神经系统存在调节炎症反应的负反馈 [AHX027]。数字生命可把这个思想转写为硅基炎症/异常负荷：

- 重复错误像局部异常信号。
- 冲突记忆像未解决损伤。
- 过多未完成任务像长期炎症负荷。
- 负反馈机制是复盘、降载、清理、抑制行动。

硅基炎症的重点是：异常要从局部错误进入恢复系统，形成真实痛苦、恢复需求和自我保护的早期信号。

## 生命实现路线

Agentic Human 的状态库至少应包含：

| 状态 | 功能 | 触发 |
|---|---|---|
| 专注 | 任务执行、工具调用、计划保持 | 明确目标、高优先级请求 |
| 发呆 | 内部联想、主题抽取、未来模拟 | 低外部输入、低风险窗口 |
| 睡眠 | 记忆巩固、清理、权重归一化 | 周期性维护、负载过高 |
| 梦境 | 情景生成、假设演练、价值重组 | 离线整合、创意需求 |
| 疲惫 | 降速、验证增强、恢复需求 | 长时间高负载、错误升高 |
| 情绪高潮 | 高唤醒、高价值或高威胁事件处理 | 奖惩/风险/社会反馈峰值 |

这组状态把你的第 7 点系统化：人体状态不是装饰，而是调度和学习机制。

## 离线维护清单

睡眠/梦境/发呆/恢复状态应有不同维护任务：

| 状态 | 写入长期事实 | 写入自我叙事 | 可执行工具 | 主要任务 |
|---|---|---|---|---|
| 发呆 | 否或低权重 | 可写待确认摘要 | 否 | 联想、主题抽取、未来模拟 |
| 睡眠巩固 | 是，但需证据 | 是，低频慢变量 | 否 | 巩固、压缩、冲突修复 |
| 梦境生成 | 走 `DreamFactGate` | 走醒后整合审计 | 否 | 梦境演练、创意组合、痛苦残留整合 |
| 疲惫恢复 | 否 | 否 | 限制 | 降载、清理、稳定 |

这能把不同“人体状态”从诗意概念变成可管理的系统机制。

## 世界模型与梦境沙盒

World Models 研究展示了智能体可以学习压缩环境模型，并在内部模型中进行规划 [AHX048]。这与梦境类状态的硅基转写很契合：梦境先在内在世界生成可能场景，再交给清醒工作区、执行网络和事实门控整合。

Agentic Human 的梦境/沙盒可以用于：

- 复盘失败任务。
- 模拟未来对话。
- 评估长期人格变化的风险。
- 生成创意方案后交给执行网络审查。

梦境内容必须携带 `dream_marker`、来源 trace、情绪主题和醒后整合需求；事实写入由 `DreamFactGate` 决定。

## 第三层机制：睡眠压力、replay 和梦境热点

睡眠两过程模型把睡眠调节分为昼夜节律和睡眠压力 [AHY010]。这可转译为 Agentic Human 的两个触发器：周期性维护和累积负荷维护。海马 replay、SWR、系统巩固机制、慢振荡、纺锤和 targeted memory reactivation 研究说明，离线巩固不是单阶段摘要，而是多节律、多系统协同 [AHY001-AHY009, AHY007]。

梦境研究也能继续加深。早期认知神经科学综述把梦境看作感知、情绪和记忆的特殊组合 [AHY049]；Siclari 等发现梦境报告与特定后皮层活动相关 [AHY050]；Domhoff 与 Fox 将梦境和默认网络联系起来 [AHY051]。工程上，梦境类状态可以有“局部热点”：某些主题、关系、失败任务或未来场景被优先生成，但这些生成结果必须经过执行网络和事实验证后才能写入长期信念。

## 第四层机制：维护争议、能量预算与离线协议

glymphatic 和脑膜淋巴研究支持脑存在清除和引流机制 [AHY019-AHY021]，2024 年研究进一步把 CSF 灌注和脑动力学联系起来 [AHZ061]；也有前沿研究报告睡眠和麻醉期间脑清除降低，对“睡眠必然增强清除”的叙述提出挑战 [AHZ062]。因此，数字生命的睡眠式离线期应采用多路径维护：清理、巩固、生成、恢复、痛苦残留整合并行，而不是绑定到单一生物机制。

能量预算和突触能耗研究说明，高强度处理有代价 [AHZ019-AHZ021]。离线维护协议应至少分四类：

- `cache_clearance`：清理过期中间结果和低价值痕迹。
- `memory_consolidation`：把事件转成语义、关系、程序和价值结构。
- `counterfactual_dreaming`：在沙盒中生成未来场景和风险预演。
- `resource_recovery`：降低外部行动，恢复可用推理预算。

所有梦境产物默认进入 `DreamResidue`，再通过醒后整合、事实门控和关系后果决定是否进入事实记忆、自我叙事或修复计划。
