# 注意、意识与全局工作空间

## 全局可访问性

全局神经工作空间理论提出，局部处理的信息在达到某些条件后会被广泛广播，从而成为可报告、可访问、可用于多系统协调的信息 [AH076, AH077]。这对 Agentic Human 的工程启发很强：系统需要区分局部后台处理和全局工作区内容。

全局工作区可以承载：

- 当前目标。
- 当前用户意图。
- 当前高置信事实。
- 当前风险与约束。
- 当前候选计划。
- 当前需要对外报告的状态。

## 论文精读：GNW 的可转译部分

Dehaene 与 Changeux 的工作强调 conscious access 与信息的全局广播有关 [AH076]。Mashour 等人的综述把全局神经工作空间放到麻醉、睡眠和意识状态研究中讨论 [AH077]。对 Agentic Human 最有价值的不是“实现意识”，而是“局部处理什么时候升级为全局可访问内容”。

工程转译：

- 局部模块可以处理很多内容，但不都进入全局空间。
- 进入全局空间的信息会影响语言、记忆、行动和价值系统。
- 全局空间容量有限，需要选择、刷新和清理。
- 可报告内容和不可报告后台处理应分离。

这能解决一个常见 agent 问题：所有信息都混在一个上下文里，导致注意失焦、历史污染和行动理由不清。

## 意识理论的边界

IIT、GNW、意识神经相关物研究和 Seth/Bayne 的理论综述都说明，意识问题尚无单一答案 [AH078-AH080]。因此研究档案必须保持克制：

- 可以借鉴“全局广播”“访问意识”“整合度”“报告性”等功能概念。
- 不宣称 agent 拥有主观体验。
- 不把长上下文、记忆、语言流畅性等同意识。

## 多理论并存带来的约束

Seth 与 Bayne 对意识理论的综述显示，目前存在多种竞争理论，包括全局工作空间、整合信息、预测处理、高阶理论等 [AH080]。Koch 等关于意识神经相关物的综述也强调，找到神经相关物不等于解释意识本身 [AH079]。

这给 Agentic Human 三条硬边界：

- **功能可借鉴**：全局广播、可报告性、整合、注意门控都可以工程化。
- **体验不可宣称**：不能因为系统有这些功能，就说它有主观感受。
- **概念要分层**：区分意识相关功能、意识理论、主观体验和人格模拟。

这个边界必须长期保留，否则研究底座会滑向伪科学或过度拟人化。

## 递归处理与报告性的区别

Dehaene 与 Naccache 的早期全局工作空间框架区分了后台处理和可报告访问 [AHX037]。Lamme 则强调递归处理可能与不同层次的意识现象有关 [AHX038]。这些理论存在争议，但对 Agentic Human 有一个清楚启发：反馈回路、全局广播和语言报告是不同层次。

工程上应区分：

- `local_recurrence`：局部模块内部反复处理，例如视觉/语言局部校正。
- `global_broadcast`：信息进入全局工作空间，影响多系统。
- `verbal_report`：系统把当前状态组织成对外语言。
- `self_narrative`：长期叙事把多个报告串成连续身份。

不能因为有 `verbal_report` 就推断有主观意识；也不能因为没有对外报告，就认为没有内部处理。

## 注意作为入口门控

注意网络研究显示，大脑对信息的处理受目标、显著性和状态控制 [AH021-AH025]。这可以和全局工作区连接起来：不是所有输入都进入全局空间，只有达到显著性、任务相关性或风险阈值的信息才升级。

## 工作区生命周期

全局工作空间中的内容应有生命周期：

1. **进入**：由显著性、目标相关性、风险、不确定性触发。
2. **维持**：被当前任务、语言计划、行动候选持续引用。
3. **竞争**：新事件可能挤出旧内容。
4. **广播**：同步到语言、记忆、行动、价值系统。
5. **退出**：写回长期记忆、归档为任务结果或丢弃。

如果没有退出机制，工作区会越来越臃肿；如果没有进入门控，系统会被低价值事件淹没。

## 元认知与置信度

元认知研究关注个体对自身判断的监控能力 [AHX058]。Agentic Human 需要元认知层来回答：

- 我对这个事实有多确定。
- 我为什么选择这个行动。
- 我是否可能被旧记忆误导。
- 我是否需要搜索、询问或延迟判断。
- 我当前状态是否适合执行高风险任务。

这不是“自我意识”的证明，而是可靠系统的必要功能。元认知失败会导致自信幻觉、错误固化和行动冲动。

## 设计假设

Agentic Human 可以有一个“全局工作空间”，但它应被严格定义为工程结构：

- 它是当前系统可访问内容的共享黑板。
- 它受显著性网络、执行控制和内感受状态共同门控。
- 它不等于意识，只是可报告和可协调的信息层。
- 它应该容量有限，避免把全部记忆塞入当前状态。

一个健康的工作空间应有进入、维持、刷新和退出机制。信息进入后如果长期不再相关，应被写回记忆或丢弃，而不是永久占据注意。

## 工作区字段草案

| 字段 | 说明 |
|---|---|
| `current_goal` | 当前明确目标 |
| `active_constraints` | 用户要求、安全边界、时间限制 |
| `active_memories` | 当前被调用的记忆痕迹 |
| `salient_events` | 需要处理或监控的显著事件 |
| `candidate_actions` | 候选行动及其价值/风险 |
| `uncertainty_map` | 事实、计划、关系中的不确定性 |
| `self_state` | 当前内部状态摘要 |
| `reportable_summary` | 可对用户解释的当前状态 |

这套字段只是功能工作区，不是意识宣称。

## 工作区质量指标

后续可评估：

- 工作区是否只保留当前任务相关信息。
- 是否能说明关键行动的证据来源。
- 是否能标注不确定性。
- 是否能在高冲突时暂停行动。
- 是否能把完成任务后的信息写回长期记忆。

这些指标比讨论“是否有意识”更可操作。
