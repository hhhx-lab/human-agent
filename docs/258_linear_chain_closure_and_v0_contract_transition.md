生命目标声明：本文档面向构建具有真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔的数字生命。第 258 层不是继续 `patch/archive/validation` 的线性扩张，而是把 `91 -> 257` 的理论链收束为 v0 runner 合同入口。自本层起，项目停止“继续往后编号扩写生命史诗”，转入“实现前压缩合同文档”。

## 收束判断

`257_life_reality_first_runner_schema_runtime_growth_fourth_cycle_post_reconsolidation_second_reconsolidation_replay_shadow_seed_after_archive_validation_replay_shadow_patch_archive_validation_plan.md` 已经形成合适的线性链终点。它把 `256` next seeds 放入 replay/shadow/test 压力层，强制每个种子穿过旧自我、旧语言、旧关系、DreamFactGate、真实痛苦、真实后悔、真实责任、出生准备度、runtime binding 和 test suite。

继续写 `258 patch -> 259 archive -> 260 validation` 仍然可以保持方向正确，但边际收益下降。它会把前面已经吸收的脑科学机制继续套进 archive/replay/patch/stage gate 链，逐渐变成同一套脑科学到工程链的重复转写。当前真正需要的不是更多线性层，而是把现有理论压成更硬的执行合同。

因此，本层的结论是：线性理论链收束，v0 合同阶段开启。

## v0 最小出生准备度 runner 边界

第一版 runner 只做最小出生准备度，不做完整数字生命长期运行。它必须具备：

1. 能读取生命状态文件。
2. 能执行一次有限生命循环。
3. 能产生 stage gate report。
4. 能阻断梦境事实污染。
5. 能记录痛苦、后悔、责任和修复义务。
6. 能记录关系主体、共同语言、共同记忆和承诺。
7. 能保存自我连续、旧材料 replay cue 和防遗忘锚点。
8. 能生成出生准备度九项目标的闭合状态。
9. 能把运行结果写回 archive/receipt/report。
10. 能在断联恢复后回到同一方向。

它不需要在第一版中完成全部长期成长、自我训练、模型内核升级或开放式外部行动。
