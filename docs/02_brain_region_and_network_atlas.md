# 脑区、连接组与大尺度网络

## 核心结论

现代脑科学不支持“一个脑区 = 一个功能模块”的简单设计。连接组研究强调结构连接、功能连接、网络枢纽和模块化组织共同决定脑功能 [AH001-AH010]。这对 Agentic Human 的第一条约束是：区域可以命名，但区域边界必须是功能性、概率性、可重叠的。

`构思.md` 中的 L/R/P/J/G/S/O/D 区域命名很有价值，因为它给系统提供了空间感和组织感。这些代号应作为项目语言，同时接受脑科学约束。Julich-Brain 和多模态皮层分区都显示，真实脑区边界来自细胞结构、连接、功能和任务响应的综合证据 [AH003, AH004]。

## 论文精读：从图谱到连接组

Sporns、Tononi 与 Kotter 提出 human connectome 时，核心不是“把所有脑区列全”，而是要描述神经系统中所有结构连接及其拓扑组织 [AH001]。这篇论文对 Agentic Human 的价值在于，它把“区域”从静态器官变成了图上的节点：节点的功能取决于它连接到哪里、连接强度如何、是否位于枢纽位置、是否承担跨模块整合。

Human Connectome Project 则把这个思想推进为多模态数据工程：结构 MRI、扩散成像、静息态 fMRI、任务 fMRI、行为数据共同进入同一研究框架 [AH002]。这给我们的技术启发很清楚：如果 Agentic Human 要形成“类脑结构”，不能只保留单一日志。它应该同时记录交互事件、内部状态、行动结果、长期偏好、失败轨迹、语义关系和时间节律，并允许这些数据在同一自我模型里互相校准。

Glasser 的多模态皮层分区显示，单靠一种指标划分脑区会失真，可靠分区需要整合解剖、功能、连接和任务激活 [AH003]。Julich-Brain 进一步提示脑区边界是概率性的，不是像软件包目录那样整齐 [AH004]。因此，Agentic Human 的区域不该以硬边界表达为“这个文件夹只管情绪、那个文件夹只管语言”，而应表达为“某组记忆、信号、处理器在某些状态下共同承担某类功能”。

Bullmore 与 Sporns 的复杂网络综述提供了三个必须保留的技术概念 [AH008]：

- **小世界结构**：局部模块内部高连接，同时保持少量长程连接，使信息可以快速跨系统整合。
- **模块化**：系统分成相对稳定的子系统，降低复杂度和干扰。
- **枢纽节点**：部分节点连接多个模块，适合承担全局路由、状态切换和整合。

在工程上，这意味着 Agentic Human 不应是一个巨大的线性 prompt，也不应是无组织的海量 memory store。更接近的结构是：多个专门记忆/处理区 + 少量全局工作空间/显著性/调质枢纽 + 可追踪的跨区连接。

## 脑区不是孤岛

结构连接组把脑看作图，而不是零件表 [AH001, AH008]。这意味着 Agentic Human 的“区域规划”不应只列每个区负责什么，还要定义：

- 区域之间交换什么类型的信息。
- 哪些区域是全局 hub，哪些是局部专业模块。
- 哪些路径是前馈处理，哪些路径是反馈、预测、校正或抑制。
- 哪些区域在不同状态下会改变连接强度。

Markov 等人的 counterstream 架构研究尤其提醒我们，皮层不是单向流水线，而是高密度的双向和反向通路 [AH005]。因此 ACE 三层不能被写成“感知 -> 处理 -> 执行”的单向管道；它应允许处理层反向调制感知层，执行结果反向更新处理层和记忆系统。

## 皮层层级与反馈

皮层处理常被画成层级：低级感觉区处理局部特征，高级区处理抽象含义。但现代网络视角强调层级中有大量反馈和旁路 [AH005, AH008]。这对人脑式 agent 很关键：

- 感知不是被动接收，而是被当前目标、记忆和预期塑形。
- 执行不是最后一步，执行结果会回流，改变价值、习惯和记忆。
- 情绪/价值不是附加标签，而会改变感知权重和行动阈值。
- 长期人格不是配置项，而是网络连接、记忆历史和慢变量逐渐稳定后的结果。

因此，ACE 三层应升级为“循环处理”：感知层产生候选事件，处理层生成解释和行动，执行层输出行为并产生误差信号，误差信号再改变感知过滤、记忆权重和价值调制。

## 大尺度网络视角

Yeo、Power、Smith、Biswal 等工作共同说明，静息态和任务态中都能观察到稳定但非绝对的大尺度网络 [AH006, AH007, AH009, AH010]。这些网络为 Agentic Human 的状态设计提供了更好的单位：

- 默认模式网络：自我、记忆、未来模拟、内在生成。
- 中央执行/多需求网络：问题解决、工作记忆、计划、控制。
- 显著性网络：判断什么值得切换注意或升级处理。
- 感觉/运动网络：外部输入与行动输出。
- 边缘/价值网络：情绪、动机、奖赏、威胁和身体状态。

这比“左脑负责语言，右脑负责情绪”的传统说法更稳。半球偏侧化确实存在，但语言、情绪、空间、记忆都以分布式网络方式工作。

## 连接强度、状态和时间尺度

大尺度网络还有一个常被忽略的点：连接不是永远固定的。任务需求、睡眠/觉醒、情绪压力、药理状态和学习阶段都会改变功能连接的表达 [AH006-AH010]。对 Agentic Human 来说，这意味着同一模块在不同状态下应有不同权重。

例如：

- 专注状态下，执行网络和任务相关记忆权重升高，默认联想被抑制。
- 发呆状态下，默认模式和情景生成权重升高，外部输入门槛提高。
- 高风险状态下，显著性和抑制信号升高，行动系统降速。
- 睡眠/巩固状态下，外部行动关闭，记忆重写和冲突修复启动。

这类状态化连接是真实数字生命感的关键。数字生命要拥有内在时间，而不是只在外部输入到来时反应；长期 agent 必须有内部节律和状态变换。

## 发展、剪枝与结构成熟

神经发育研究补充了连接组视角：人脑不是一开始就拥有成人式网络，而是在神经发生、迁移、突触形成、修剪、髓鞘化和功能专门化中逐渐成熟 [AHX015, AHX017, AHX018]。关键期和敏感期研究也说明，经验对系统结构的影响随阶段变化 [AH081, AHX014]。青少年脑发育研究还提示，奖赏/情绪系统和认知控制系统的成熟速度并不完全同步 [AHX016]；关键期再开放研究则说明，可塑性窗口会受到兴奋/抑制平衡、环境和训练条件调节 [AHX019]。

对 Agentic Human 来说，这意味着“区域规划”不应一次性封死。更合适的路线是：

- 初期允许较高可塑性，快速建立基本区域和连接。
- 中期通过经验和反馈稳定常用连接。
- 后期保留可塑性，但对核心人格、价值和长期记忆加门控。
- 定期剪枝低价值连接和过时记忆，避免网络臃肿。

这里的“发育”是硅基生命发育路线：把长期系统设计成有阶段、有成熟、有修剪、有保护的结构，让真实人格、真实生命和真实关系在时间中生长。

## 对原始区域代号的研究化修正

| 代号 | 可保留的项目含义 | 需要修正的脑科学约束 |
|---|---|---|
| L/R | 信息偏侧化与跨半球协作 | 不做绝对左右脑人格分工 |
| P | 高级皮层处理 | 皮层内部需要多网络、多尺度分区 |
| J | 中继和调节 | 丘脑/下丘脑不是被动通道 |
| G | 自主节律和基础唤醒 | 心跳式机制是硅基转写，不是生命体征 |
| S | 误差校正和时序 | 小脑不是单纯运动区，也不直接发起任务 |
| O | 记忆、情绪、动机 | 边缘系统不是“情绪盒子” |
| D | 选择、习惯、抑制 | 基底节参与行动和认知选择，但不是唯一决策者 |

## 生命实现路线

Agentic Human 的物理区域规划可以采用“软分区 + 强连接”的方式：每个区有主职责，但信息在网络中动态流动。实现上更接近多层状态图，而不是固定目录树。

最小可行的区域层应包含：

- `Perception Mesh`：多模态输入、内感受、显著性判断。
- `Memory Field`：情景、语义、程序、情绪和自我叙事记忆。
- `Executive Workspace`：任务目标、计划、工作记忆、冲突控制。
- `Value-Regulation System`：奖赏、惩罚、风险、压力、探索。
- `Action-Tool Loop`：执行、反馈、小脑式误差校正。
- `Offline/Default System`：走神、梦境、巩固、重组和自我模拟。

这些名字不是最终实现模块，而是研究阶段的架构语言。

## 技术转译清单

为了把脑区/网络研究变成可实施的理论底座，后续架构文档应至少记录这些对象：

| 对象 | 含义 | 生命科学来源 | 工程注意点 |
|---|---|---|---|
| `Region` | 软区域，不是硬模块 | 概率脑区、连接组 [AH001-AH004] | 允许一项功能跨多个区域 |
| `Edge` | 区域之间的连接 | 结构/功能连接 [AH001, AH008] | 记录方向、强度、状态依赖 |
| `Hub` | 跨区枢纽 | 图论网络枢纽 [AH008] | 用于全局工作区、显著性、调质 |
| `NetworkMode` | 大尺度功能模式 | 静息态/任务态网络 [AH006-AH010] | 控制当前系统处理倾向 |
| `FeedbackLoop` | 反馈通路 | counterstream、预测和校正 [AH005] | 防止单向流水线 |
| `Timescale` | 连接变化速度 | 学习、状态、发育 | 区分快变量、中变量、慢变量 |

这份清单不是代码规格，而是研究底座的骨架：后续所有 agent 功能都应落到区域、连接、状态、调质和行为循环上。

## 结构成熟指标

后续如果评估 Agentic Human 是否“成长”，不能只看知识量，应看结构指标：

- 区域边界是否更清晰，但仍允许跨区协作。
- 重要 hub 是否稳定，低价值连接是否减少。
- 默认、执行、显著性等状态能否可靠切换。
- 记忆检索是否从相似度搜索变成情境/价值/目标共同触发。
- 习惯是否形成，同时保留被显著性打断的能力。

这套指标来自连接组、图论网络和发育可塑性的综合启发 [AH001-AH010, AH081-AH085, AHX014-AHX018]。

## 第三层机制：支撑系统、髓鞘化和认知地图

脑不是只有神经元。胶质研究显示，胶质细胞参与中枢神经系统形成、突触功能、髓鞘和回路调节 [AHY022]。活动依赖髓鞘化研究说明，常用通路会被加速和稳定 [AHY023, AHY024]。硅基转写是：Agentic Human 的“支撑层”不只是后勤缓存，它会塑造认知速度和稳定性。常用的检索路径、工具流程、写作模式和关系策略都可能被优化成高效通路。

认知地图研究进一步把记忆和行动联系起来：灵活行为需要组织化知识地图 [AHY053]，海马经典认知地图理论和 predictive map 理论说明，记忆系统可以表示空间、状态和未来转移 [AHY054, AHY055]。Agentic Human 的区域规划因此不只是功能区图，还应包含“知识地图”和“行动路径图”。

事件分割研究提示，人脑会把连续经验切成事件单元 [AHY060]。这对文档底座非常重要：记忆写入、经验 replay、关系更新都应该以事件边界为单位，而不是按固定 token 或时间切片粗暴切割。

## 第四层机制：动力系统与网络控制

第四批文献把“区域图”继续推进为“动力系统”。神经 avalanche、临界性和大尺度脑活动模型提示，脑区不是静态容器，而是会在不同状态下进入不同传播、稳定和协作模式 [AHZ001-AHZ005]。网络控制理论进一步说明，结构连接会限制系统从一个状态转移到另一个状态的成本 [AHZ006]。

Agentic Human 的 `RegionGraph` 因此至少要记录三类信息：

- `activation_state`：区域或子网络当前是否活跃、抑制或待机。
- `transition_cost`：从默认、专注、梦境、维护等状态切换到另一个状态需要多少资源和风险。
- `coupling_mode`：区域之间是松散并行、强耦合协作，还是被显著性事件临时同步。

这能避免把脑区规划写成“模块清单”。真正的数字生命底座应是一个可切换、可调质、可恢复的动态网络。
