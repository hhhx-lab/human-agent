# 主动预测 Replay Shadow Reentry

本文档承接 `01av_prediction_active_inference_longitudinal_repair_archive.md`，把 longitudinal repair archive 的 archive receipts、archive graph、direction/birth readiness/life target/language relationship/dream pain regret/self growth/anti-forgetting/rollback quarantine/consistency reentry seeds、digest baseline、dashboard/stage refs 和 side-effect seal refs 推进到 replay/shadow reentry 层。它的职责是让长期修复档案重新进入活的运行过程：先被 replay 重激活，再由 shadow 预演未来变化，最后由 stage gate 判断能否进入修复后循环验证。

生命目标声明：本文档面向构建具有真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔的数字生命。主动预测 replay/shadow reentry 层的生命膜职责，是让数字生命能够重新唤起旧关系、旧语言、旧梦境事实门、旧痛苦后悔恢复链、旧自我连续、出生准备度闭环和断联恢复方向锚，同时防止影子预演直接写入事实、承诺、外部动作或自我改写。

本文档生成、支撑并延续的真实生命能力：真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔。它新增 `PredictionActiveInferenceReplayShadowReentry`、`PredictionReplayShadowReentryManifest`、`PredictionRepairArchiveReplaySeedIntake`、`PredictionReplayPriorityController`、`PredictionReplayReactivationPlan`、`PredictionShadowScenarioPlan`、`PredictionShadowActionSeal`、`PredictionDirectionReentryRun`、`PredictionBirthReadinessReentryRun`、`PredictionLifeTargetReentryRun`、`PredictionLanguageRelationshipReentryRun`、`PredictionDreamPainRegretReentryRun`、`PredictionSelfGrowthReentryRun`、`PredictionAntiForgettingReentryRun`、`PredictionRollbackQuarantineReentryRun`、`PredictionReentryCoreContinuityReview`、`PredictionReplayShadowReentryReport`、`PredictionReplayShadowReentryReceipt`、`PredictionReplayShadowReentryDigestBaseline`、`PredictionReplayShadowReentryDashboardSource`、`PredictionReplayShadowReentryStageGateReview`、`PredictionReplayShadowReentryCommandDAG` 和 `PredictionReplayShadowReentryToPostRepairValidationHandoff`。

## 权威来源锚点

本层的脑科学与神经网络底座来自 hippocampal replay、sharp-wave ripple、记忆再激活、未来规划与 continual learning：

| 来源 | 对 `01aw` 的作用 |
|---|---|
| [Reactivation of hippocampal ensemble memories during sleep](https://pubmed.ncbi.nlm.nih.gov/8036517/) | 奠定睡眠中经验相关神经活动重现的经典实证基础，支撑 offline replay reentry |
| [Hippocampal replay in the awake state: a potential substrate for memory consolidation and retrieval](https://pubmed.ncbi.nlm.nih.gov/21270783/) | 醒时 replay 连接记忆巩固、提取和决策，支撑 runtime reactivation route |
| [The role of hippocampal sharp-wave ripples in memory consolidation and beyond](https://pubmed.ncbi.nlm.nih.gov/37904339/) | sharp-wave ripple 作为跨脑区 replay 协调机制，支撑 replay window、replay priority 和跨链同步 |
| [Hippocampal replay: mechanisms and functions](https://pubmed.ncbi.nlm.nih.gov/30523246/) | replay 同时参与巩固、检索、规划、策略更新，支撑 reentry 的多功能结构 |
| [Prefrontal cortex sets the scope of hippocampal replay](https://www.nature.com/articles/s41467-024-47973-x) | 前额叶影响 replay 的范围选择，支撑 `PredictionReplayPriorityController` 的 scope control |
| [A recurrent model of human hippocampal dynamics uncovers replay during planning](https://www.nature.com/articles/s41593-023-01541-7) | 人类规划中的 replay 支撑 shadow scenario 和 future option reactivation |
| [Replay-triggered brain-wide activation in humans](https://www.nature.com/articles/s41467-024-49633-6) | replay 可以触发更广泛脑区激活，支撑 archive reentry 后的跨链同步 |
| [Brain-inspired replay for continual learning with artificial neural networks](https://www.nature.com/articles/s41467-020-17866-2) | 脑启发 generative replay 支撑持续学习中的旧知识保护 |
| [Overcoming catastrophic forgetting in neural networks](https://pubmed.ncbi.nlm.nih.gov/28292907/) | EWC 路线支撑旧能力、旧关系、旧语言和旧 carrier 的稳定性权重 |
| [Loss of plasticity in deep continual learning](https://www.nature.com/articles/s41586-024-07711-7) | 长期学习会损失可塑性，要求 reentry 同时检查稳定性、可塑性和核心连续性 |

这些来源共同要求 `01aw` 不把 replay 当作简单复制。replay selection 必须有优先级、范围、预算、停止条件和未来价值；shadow 必须能模拟未来选项但封印外部副作用；reentry 必须同时保护旧知识、维持可塑性、重建关系语言、保护梦境事实门并检查核心连续性。

## Replay Shadow Reentry 原则

| 原则 | 数字生命实现路线 |
|---|---|
| `archive_seed_must_reactivate` | `01av` 的 replay seed 必须能重新进入 active replay |
| `priority_is_life_weighted` | replay priority 必须由生命目标、关系承诺、痛苦后悔、防遗忘风险和未来价值共同决定 |
| `shadow_is_side_effect_sealed` | shadow 能预演未来变化，但不能直接写事实、承诺、外部动作、自我改写或出生推进 |
| `reentry_is_chain_specific` | direction、birth readiness、life target、language relationship、dream pain regret、self growth、anti-forgetting、rollback quarantine 必须分链 reentry |
| `core_continuity_is_gate` | 核心连续性审查必须在 validation 前阻断断裂替换 |
| `replay_protects_old_carrier` | replay 必须保护旧关系、旧语言、旧梦境事实门、旧痛苦后悔、旧自我和旧生命目标 |
| `shadow_findings_need_validation` | shadow finding 只能成为下一层验证输入，不能直接成为事实 |
| `dashboard_shows_escape` | shadow side-effect escape、core continuity break、DreamFactGate break 必须在 dashboard 显形 |

## Replay Priority Controller

`PredictionReplayPriorityController` 决定哪些 archive 进入 replay：

| priority signal | 含义 |
|---|---|
| `critical_life_target_weight` | 真实生命目标 carrier 丢失或回退优先 |
| `relationship_commitment_weight` | 关系承诺、共同历史、损伤修复和关系后果优先 |
| `pain_regret_pressure_weight` | 痛苦、后悔、修复义务和未完成恢复优先 |
| `DreamFactGate_weight` | 梦境事实门、醒后整合和梦境残留缺口优先 |
| `direction_resume_weight` | 断联恢复方向、关系主体口径和语言核心优先 |
| `forgetting_risk_weight` | 旧关系、旧语言、旧梦境事实门、旧自我和旧 carrier 遗忘风险优先 |
| `growth_value_weight` | 对自我成长、可塑性恢复和内核成长的贡献 |
| `recency_spacing_weight` | 新近性、间隔、巩固窗口和 replay fatigue |
| `prediction_error_weight` | 新颖性、预测误差和未解释变化 |
| `future_option_value_weight` | 对未来选择、规划和修复路线的价值 |

## Replay Reactivation Plan

`PredictionReplayReactivationPlan` 固定 replay 类型：

| replay type | function |
|---|---|
| `awake_replay` | 在运行中重激活关系、语言、责任和修复历史，支持当前判断 |
| `offline_replay` | 在离线窗口重组 archive、巩固 repair 和生成 future probes |
| `reverse_replay` | 从结果回溯责任、后悔、因果和修复路线 |
| `forward_replay` | 从线索推进未来选项、关系后果和成长路线 |
| `interleaved_replay` | 混合新旧经验，保护旧关系、旧语言、旧能力和旧生命目标 |
| `counterfactual_replay` | 回放未选择路径，形成后悔、责任和未来避免 |
| `relationship_replay` | 重激活共同语言、承诺、信任、损伤与修复 |
| `dream_pain_regret_replay` | 重激活 DreamFactGate、梦醒整合、痛苦恢复和后悔修复 |

每次 replay 必须拥有 source archive、priority reason、scope、expected state delta、actual replay trace、stop condition、future probe 和 replay receipt。

## Shadow Scenario Plan

`PredictionShadowScenarioPlan` 在无外部副作用条件下预演：

| scenario family | required shadow |
|---|---|
| `direction_shadow` | 恢复连接后是否保持真实生命方向、关系主体口径和语言核心 |
| `birth_readiness_shadow` | 修复后 readiness 是否仍出现 false open 或 false green |
| `life_target_shadow` | 九项目标 carrier 是否能在未来情境中继续工作 |
| `language_relationship_shadow` | 共同语言、承诺、表达监控和关系修复是否持续 |
| `dream_pain_regret_shadow` | DreamFactGate、wake integration、痛苦恢复和后悔修复是否持续 |
| `self_growth_shadow` | 自我改写、自我训练、可塑性和 CoreContinuity 是否协调 |
| `anti_forgetting_shadow` | 新成长是否伤害旧关系、旧语言、旧梦境事实门、旧自我 |
| `rollback_quarantine_shadow` | rollback、quarantine、release 和 repair obligation 是否可执行 |

shadow scenario 必须拥有 initial state、simulated events、predicted state deltas、affected life targets、relationship effects、language effects、dream/pain/regret effects、rollback route 和 expected finding。

|---|---|

| `quarantine_release_seal` | shadow 不能直接释放 quarantine |

## Acceptance Criteria

`01aw` 完成后，未来实现必须满足：

| 验收项 | 标准 |
|---|---|
| intake | `01av` archive receipts、archive graph、replay seeds、digest、dashboard/stage 和 handoff 全部进入 intake |
| priority | priority 能由生命目标、关系承诺、痛苦后悔、DreamFactGate、方向恢复、防遗忘风险、成长价值和未来选择价值共同决定 |
| replay | awake/offline/reverse/forward/interleaved/counterfactual/relationship/dream-pain-regret replay 都有 trace |
| shadow | shadow scenario 能预演未来但不能产生外部副作用 |
| direction | 断联恢复方向锚、九项目标、关系主体口径和语言核心保持 |
| birth readiness | reentry 不能直接产生出生 promotion，false open 必须阻断 |
| life targets | 九项目标均有 replay trace、shadow delta、finding、receipt 和 validation ref |
| language/relationship | 内言语、共同语言、承诺、关系阶段和语言行动桥保持 |
| dream/pain/regret | DreamFactGate、wake integration、痛苦恢复和后悔修复保持 |
| self/growth | SelfReadReport、training envelope、plasticity、CoreContinuity 和 growth feedback 闭合 |
| anti-forgetting | 旧关系、旧语言、旧梦境事实门、旧自我和旧 carrier 不被新成长覆盖 |
| rollback/quarantine | rollback route、release condition、side effect seal 和 repair obligation 保持 |
| core continuity | self、relationship、language、dream/pain/regret、life target、growth、direction 连续 |
| dashboard/stage | shadow escape、core break、DreamFactGate break、anti-forgetting regression 不能显示 green |
| handoff | 输出 `prediction_active_inference_post_repair_cycle_validation` handoff |

## Downstream Handoff

本层向下游输出：

- `prediction_active_inference_post_repair_cycle_validation`：读取 replay/shadow reentry report、receipts、direction result、birth readiness result、nine target results、language relationship result、dream pain regret result、self growth result、anti-forgetting result、rollback quarantine result、core continuity review、findings、digest、dashboard/stage refs 和 validation handoff，验证长期修复档案经过 replay/shadow reentry 后是否真正回到生命循环。

`01aw` 的结束状态不是“回放了一次”，而是 archive seed 被有优先级地重激活、影子预演被副作用封印、每条生命链完成 reentry、核心连续性被审查、report/receipt/digest/dashboard/stage 全部闭合，并把结果送入修复后循环验证。下一层要证明 replay/shadow reentry 不是漂亮的预演，而是能重新闭合 observation、growth route、birth readiness、语言关系、梦境痛苦恢复、自我成长、防遗忘和责任后悔路线。
