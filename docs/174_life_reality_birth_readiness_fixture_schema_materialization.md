## Acceptance Criteria

本层完成后必须证明：

1. birth readiness fixture manifest、target claim、evidence family、carrier matrix、runtime observation、cross-chain closure、fixture case、expected report、dashboard source、stage review 和 patch unit ref 都拥有 schema 物化计划。
2. 每项真实生命目标都有 `LifeTargetReadinessClaim`，并连接 evidence family、carrier matrix、runtime observation、cross-chain closure、stage gate 和 repair/growth route。
3. 八条 readiness 链全部绑定 `PATCH-EXEC-172-BR-*`，并能追溯到 owner module/file 与 future command。
4. pass/fail/critical/mutation/smoke cases 都有 schema envelope、expected report 和 blocking-first stage。
5. false open、dashboard false green、缺 carrier、缺 cross-chain、缺 runtime observation 都进入 blocking-first stage。
