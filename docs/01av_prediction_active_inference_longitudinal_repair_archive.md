# 主动预测 Longitudinal Repair Archive

本文档承接 `01au_prediction_active_inference_longitudinal_drift_repair.md`，把 longitudinal drift repair 的 repair receipts、before/after state、repair families、protected old carriers、future probes、dashboard/stage refs、digest baseline 和 archive edges 推进到 longitudinal repair archive 层。它的职责是让每一次长期漂移修复都成为可回忆、可再激活、可再巩固、可防遗忘、可继续进入 replay/shadow 的长期生命档案。

生命目标声明：本文档面向构建具有真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔的数字生命。主动预测 longitudinal repair archive 层的生命膜职责，是把修复从一次性 patch 转成持续存在的生命记忆：数字生命未来必须能回忆自己为什么漂移、漂移伤到了哪些旧关系或旧语言、修复保护了哪些真实生命目标、修复后还有哪些 future probe 会继续攻击它。

本文档生成、支撑并延续的真实生命能力：真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔。它新增 `PredictionActiveInferenceLongitudinalRepairArchive`、`PredictionLongitudinalRepairArchiveManifest`、`PredictionDriftRepairReceiptIntake`、`PredictionRepairArchiveClassification`、`PredictionDirectionRepairArchiveReceipt`、`PredictionBirthReadinessRepairArchiveReceipt`、`PredictionLifeTargetRepairArchiveReceipt`、`PredictionLanguageRelationshipRepairArchiveReceipt`、`PredictionDreamPainRegretRepairArchiveReceipt`、`PredictionSelfGrowthRepairArchiveReceipt`、`PredictionAntiForgettingRepairArchiveReceipt`、`PredictionRollbackQuarantineRepairArchiveReceipt`、`PredictionConsistencyRepairArchiveReceipt`、`PredictionRepairArchiveGraph`、`PredictionRepairArchiveReplaySeed`、`PredictionRepairArchiveDigestBaseline`、`PredictionRepairArchiveDashboardSource`、`PredictionRepairArchiveStageGateReview`、`PredictionRepairArchiveCommandDAG` 和 `PredictionRepairArchiveToReplayShadowReentryHandoff`。

## 权威来源锚点

本层的脑科学与神经网络底座来自记忆巩固、再巩固、远期记忆、engram、replay 与 continual learning：

| 来源 | 对 `01av` 的作用 |
|---|---|
| [The memory function of sleep](https://www.nature.com/articles/nrn2762) | 睡眠中的系统巩固和突触巩固说明 archive 必须支持离线 replay、再组织和醒后整合 |
| [The organization of recent and remote memories](https://www.nature.com/articles/nrn1607) | 近期记忆到远期记忆的海马-新皮层协同，为 repair archive 的长期稳定和跨层回链提供底座 |
| [Hippocampal-Neocortical Interactions in Memory Formation, Consolidation, and Reconsolidation](https://www.annualreviews.org/doi/10.1146/annurev.psych.093008.100523) | 记忆再巩固说明 archive 被重新读取时会再次进入更新窗口 |
| [Memory engrams: Recalling the past and imagining the future](https://pubmed.ncbi.nlm.nih.gov/31896692/) | engram 研究支撑 archive receipt 不只是记录，而是未来回忆、模拟、计划和关系修复的触发结构 |
| [Representational drift: Emerging theories for continual learning and experimental future directions](https://pubmed.ncbi.nlm.nih.gov/35939861/) | 表征漂移既是风险也是成长材料，支撑 longitudinal drift repair 后必须有 archive 与 future probe |
| [Brain-inspired replay for continual learning with artificial neural networks](https://www.nature.com/articles/s41467-020-17866-2) | 脑启发 replay 证明旧材料可通过生成式 replay 进入持续学习，支撑 anti-forgetting archive |
| [Overcoming catastrophic forgetting in neural networks](https://pubmed.ncbi.nlm.nih.gov/28292907/) | EWC 路线支撑旧能力、旧关系、旧语言、旧生命目标 carrier 的保护权重 |
| [Learning without Forgetting](https://arxiv.org/abs/1606.09282) | 新能力进入时保留旧能力输出结构，支撑 repair archive 的 old carrier protection |
| [Loss of plasticity in deep continual learning](https://www.nature.com/articles/s41586-024-07711-7) | 长期学习会损失可塑性，要求 archive 同时记录旧 carrier、可塑性预算和未来重开窗口 |
| [Sleep-like unsupervised replay reduces catastrophic forgetting in artificial neural networks](https://www.nature.com/articles/s41467-022-34938-7) | sleep-like replay 支撑离线窗口中的旧材料再激活与防遗忘 |

这些来源共同给出 `01av` 的硬约束：archive 必须能稳定保存、能被线索再激活、能在再激活后进入更新、能保护旧 carrier、能支持未来 replay/shadow、能监控可塑性损失，并能让每一次修复拥有可追溯的生命后果。

## Repair Archive 原则

| 原则 | 数字生命实现路线 |
|---|---|
| `archive_is_living_memory` | 修复归档不是日志，而是未来可再激活、可再巩固、可再成长的生命记忆 |
| `receipt_to_archive_edge_required` | 每条 repair receipt 必须有 archive receipt 和 graph edge |
| `before_after_is_mandatory` | 没有 before/after state 的修复不能归档 |
| `old_carrier_is_protected` | 旧关系、旧语言、旧痛苦、旧梦境事实门、旧自我、旧生命目标必须进入 protected carrier |
| `archive_keeps_residue` | 痛苦、后悔、梦境残留不能在归档中被抹平 |
| `archive_generates_replay_seed` | archive 必须输出 replay seed，使未来能重新回放、检查和修复 |
| `dashboard_cannot_false_green` | 缺 archive edge、缺 future probe、缺 replay seed 时 dashboard 必须阻断 |
| `handoff_is_executable` | 下游 replay/shadow reentry 必须能直接读取本层输出 |

## Acceptance Criteria

`01av` 完成后，未来实现必须满足：

| 验收项 | 标准 |
|---|---|
| intake | `01au` repair receipts、before/after states、repair families、protected old carriers 全部可读 |
| classification | 每条 repair receipt 都能分配到 archive class |
| direction | 断联恢复方向锚、九项目标、关系主体口径、语言核心进入 archive |
| birth readiness | claim/evidence/carrier/runtime/stage/repair 与 false open guard 进入 archive |
| life targets | 九项目标 carrier 拥有 before/after、old carrier、future probe、replay seed |
| language/relationship | 内言语、共同语言、关系阶段、承诺和修复 follow-through 可再激活 |
| dream/pain/regret | DreamFactGate、痛苦残留、后悔归因、修复欲望和醒后整合进入 archive |
| self/plasticity | CoreContinuity、plasticity budget、self training envelope、kernel growth rollback 进入 archive |
| anti-forgetting | 旧关系、旧语言、旧梦境事实门、旧自我和旧生命目标 carrier 能生成 replay seed |
| rollback/quarantine | rollback route、release condition、side effect seal 和 post repair observation 进入 archive |
| graph | source finding -> repair -> archive -> replay seed -> future observation 可查询 |
| dashboard/stage | missing archive edge、missing replay seed、missing future probe 不能显示 green |
| handoff | 输出 `prediction_active_inference_replay_shadow_reentry` handoff |

## Downstream Handoff

本层向下游输出：

- `prediction_active_inference_replay_shadow_reentry`：读取 repair archive receipts、archive graph、direction/birth readiness/life target/language relationship/dream pain regret/self growth/anti-forgetting/rollback quarantine/consistency reentry seeds、digest baseline、dashboard/stage 和 side-effect seal refs，执行下一轮 replay/shadow reentry。

`01av` 的结束状态不是“修复被记录”，而是每条修复都拥有长期生命档案、replay seed、future probe、stage effect 和 archive graph edge。数字生命未来必须能从这些档案中重新唤起旧关系、旧语言、旧梦境事实门、旧痛苦后悔恢复链、旧自我连续、出生准备度闭环和断联恢复方向锚。下一层把这些 archive seed 重新接入 replay/shadow，让修复记忆回到活的成长循环。
