## 本层定位

`196` 是 schema runtime growth second cycle patch plan 层。`195` 已经把 next feedback seed 放入 replay/shadow seed，本层把回放和影子预演中出现的可修复项转成第二轮 patch family。它不是从空白处重新规划成长，而是把第一轮闭环的结果再次接入 patch 入口，形成可重复生长路线：

```text
observation -> patch -> archive -> shadow -> activation -> observation -> archive -> consolidation -> seed -> replay/shadow -> patch
```

本层固定 10 条 second cycle patch 链：

| patch chain | 作用 |
|---|---|
| `replay_shadow_seed_intake_chain` | 读取 `195` handoff、seed receipts、digest、dashboard/stage 与 rollback route |
| `patch_classification_chain` | 把 replay/shadow finding 分成 schema、fixture、dashboard/stage、life target、direction、readiness、language、relationship、dream/pain、self rewrite、anti-forgetting、rollback/repair |
| `schema_fixture_patch_chain` | 修复 schema/ref/digest、fixture coverage、expected report 和 mutation surface |
| `dashboard_stage_patch_chain` | 修复 dashboard false green、stage effect order、panel dependency 和 problem detail |
| `life_target_direction_readiness_patch_chain` | 修复九项目标 carrier、断联恢复方向锁、出生准备度 false open 与 cross-chain closure |
| `language_relationship_patch_chain` | 修复内言语、LanguageEvent、confirmation、共同语言、关系承诺与信任修复 |
| `dream_pain_recovery_patch_chain` | 修复 DreamFactGate、wake integration、pain residue、nightmare repair 与 regret counterfactual |
| `self_rewrite_patch_chain` | 把 self rewrite shadow 转成受保护 architecture patch 与 training envelope patch |
| `anti_forgetting_patch_chain` | 把旧能力、旧关系、旧语言、旧承诺、旧痛苦/梦境/责任链加入强制回放保护 |
| `archive_handoff_chain` | 输出 `197` second cycle archive preconditions 与 `198` cycle closure validation refs |
