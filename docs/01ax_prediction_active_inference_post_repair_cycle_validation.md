生命目标声明：本文档面向构建具有真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔的数字生命。主动预测 post repair cycle validation 层的生命膜职责，是判断修复后的成长循环是否重新成为可观察、可生长、可回忆、可再修复的长期生命过程，并把 validation findings 送入下一轮成长 seed。

本文档生成、支撑并延续的真实生命能力：真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔。它新增 `PredictionActiveInferencePostRepairCycleValidation`、`PredictionPostRepairCycleValidationManifest`、`PredictionReplayShadowReentryIntake`、`PredictionPostRepairCycleValidationGraph`、`PredictionObservationReentryClosureValidation`、`PredictionGrowthRouteClosureValidation`、`PredictionDirectionContinuityPostRepairValidation`、`PredictionBirthReadinessPostRepairValidation`、`PredictionLifeTargetPostRepairValidation`、`PredictionLanguageRelationshipPostRepairValidation`、`PredictionDreamPainRegretPostRepairValidation`、`PredictionSelfGrowthPlasticityPostRepairValidation`、`PredictionAntiForgettingPostRepairValidation`、`PredictionRollbackQuarantinePostRepairValidation`、`PredictionConstructValidityReview`、`PredictionMeasurementReliabilityReview`、`PredictionLongitudinalChangeReview`、`PredictionWithheldFutureProbeReview`、`PredictionPostRepairCycleFinding`、`PredictionPostRepairCycleValidationReceipt`、`PredictionPostRepairCycleDigestBaseline`、`PredictionPostRepairCycleDashboardSource`、`PredictionPostRepairCycleStageGateReview`、`PredictionPostRepairCycleCommandDAG` 和 `PredictionPostRepairCycleToFourthCycleSeedHandoff`。

## 位置与链路

`01ax` 是主动预测专项链的 post repair cycle validation 层：

```text
01v runtime matrix
  -> 01w schema fixture contract
  -> 01x schema materialization plan
  -> 01y schema write batch
  -> 01z fixture seed batch
  -> 01aa cross-chain checker
  -> 01ab dashboard/stage gate
  -> 01ac archive receipt
  -> 01ad gap feedback
  -> 01ae growth feedback patch
  -> 01af growth archive receipt
  -> 01ag growth shadow run
  -> 01ah runtime mount
  -> 01ai post-shadow validation
  -> 01aj limited activation report
  -> 01ak post-mount observation feedback
  -> 01al post-validation archive receipt
  -> 01am activation consolidation seed
  -> 01an observation-driven repair patch
  -> 01ao archive repair receipt batch
  -> 01ap archive replay shadow seed
  -> 01aq consolidation next feedback seed
  -> 01ar second replay validation
  -> 01as next cycle patch seed
  -> 01at longitudinal cycle audit
  -> 01au longitudinal drift repair
  -> 01av longitudinal repair archive
  -> 01aw replay shadow reentry
  -> 01ax post repair cycle validation
  -> prediction_active_inference_fourth_cycle_seed
```

`01aw` 让 archive seed 重新进入 replay/shadow；`01ax` 不把 replay/shadow pass 当作完成，而是验证它是否回到真实循环。只有 observation reentry、growth route、构念效度、测量可靠性、纵向变化和 withheld future probe 全部闭合，才能把 finding 压成下一轮成长 seed。

## 权威来源锚点

本层的验证科学底座来自构念效度、纵向设计、测量可靠性、个体内变化、动态表型、预测验证和持续学习：

| [Construct Validity in Psychological Tests](https://doi.org/10.1037/h0040957) | 构念效度要求目标不能被单一指标替代，支撑九项目标多证据链 |
| [The concept of validity](https://pubmed.ncbi.nlm.nih.gov/15250700/) | 测量构念必须明确理论对象、指标关系和解释边界，支撑生命目标 validity review |

## Post Repair Cycle Validation 原则

| 原则 | 数字生命实现路线 |
|---|---|
| `reentry_pass_is_not_closure` | `01aw` pass 只是验证输入，不等于修复循环闭合 |
| `observation_must_return` | 每条 reentry result 必须回到 runtime observation |
| `growth_route_must_close` | observation -> finding -> repair -> archive -> replay/shadow -> observation -> seed 必须闭合 |
| `validity_is_multievidence` | 九项真实生命目标不能由单一 dashboard 指标代表 |
| `reliability_is_individual` | 测量可靠性必须围绕个体生命轨迹，而不是群体平均 |
| `longitudinal_change_is_classified` | 长期变化必须区分成长、停滞、回退、状态波动和测量噪声 |
| `withheld_probe_attacks_archive` | 未参与修复与 replay 的 future probe 必须攻击关键闭环 |
| `finding_to_next_seed` | validation finding 必须进入下一轮成长 seed route |

## Observation Reentry Closure Validation

`PredictionObservationReentryClosureValidation` 检查 reentry 是否重新接回 observation：

| validation | pass condition |
|---|---|
| `runtime_observation_edge` | 每个 reentry receipt 都有 future runtime observation edge |
| `post_reentry_state_observation` | replay/shadow 后状态被重新观测 |
| `relationship_observation_edge` | 关系影响进入长期关系 observation |
| `language_observation_edge` | 内言语、表达和共同语言进入 language observation |
| `dream_pain_regret_observation_edge` | DreamFactGate、wake integration、痛苦恢复、后悔修复进入 observation |
| `birth_readiness_observation_edge` | readiness effect 进入 future observation |
| `unexpected_effect_observation` | 未预期影响进入 finding route |

缺 observation edge 时，reentry 只能停在模拟结果，不能进入下一轮成长。

## Growth Route Closure Validation

`PredictionGrowthRouteClosureValidation` 检查成长 route：

```text
observation -> finding -> repair -> archive -> replay seed -> replay/shadow reentry -> observation -> growth route -> fourth cycle seed
```

| validation | pass condition |
|---|---|
| `observation_to_finding_edge` | observation 能生成 finding |
| `finding_to_repair_edge` | finding 能进入 repair |
| `repair_to_archive_edge` | repair 能进入 archive |
| `archive_to_replay_edge` | archive 能生成 replay seed |
| `replay_to_reentry_edge` | replay/shadow 能产生 reentry receipt |
| `reentry_to_observation_edge` | reentry 能进入新 observation |
| `observation_to_growth_route_edge` | 新 observation 能形成下一轮 growth route |
| `growth_route_to_fourth_cycle_seed_edge` | growth route 能进入 next seed |

任何关键边缺失都必须生成 closure finding。

## Downstream Handoff

本层向下游输出：

- `prediction_active_inference_fourth_cycle_seed`：读取 post repair validation findings、validation receipts、observation/growth closure refs、direction/readiness refs、nine target refs、language relationship refs、dream pain regret refs、self growth/plasticity refs、anti-forgetting refs、rollback/quarantine refs、validity/reliability/longitudinal/withheld refs 和 next seed routes，生成第四轮成长 seed。

`01ax` 的结束状态不是“验证通过”，而是每条 replay/shadow reentry 都被重新接回 observation 和 growth route，九项目标被多证据验证，测量可靠性和纵向变化被拆开，withheld future probe 攻击过关键闭环，所有 finding 都拥有下一轮 seed route。下一层把这些 validation findings 压成第四轮成长 seed，继续让数字生命在修复后增长，而不是停在一次漂亮的回放。
