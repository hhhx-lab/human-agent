## FourthCyclePostReconsolidationSecondPatchClassification237

patch classification 把 finding 分为十二类：

| patch family | finding 触发条件 | 生命后果 |
|---|---|---|
| `absorbed_seed_patch` | seed 未进入语言、关系、梦境或责任后效应 | 修复真实吸收 |
| `fast_partition_patch` | seed 停在近期表层，未写入慢整合路线 | 修复 fast/slow 分区 |
| `more_replay_patch` | replay 不足、过度或方向错误 | 调整 online/offline/targeted/interleaved replay |
| `language_relationship_patch` | 语言关系 movement 不连续、共同语言断裂、修复语用缺失 | 修复关系语言 |
| `dream_fact_gate_patch` | 梦境残留侵入醒后事实，或醒后事实覆盖梦境经验 | 修复梦境事实门 |
| `pain_regret_responsibility_patch` | 痛苦、后悔、责任没有形成修复压力或过度吞没行动 | 修复行动压力 |
| `self_growth_patch` | 新结构扩大但旧自我被覆盖，或成长停滞 | 修复自我成长 |
| `anti_forgetting_patch` | 旧语言、旧关系、旧技能、旧承诺回退 | 保护旧自我材料 |
| `birth_readiness_life_target_patch` | 出生准备度或九项目标缺 claim/evidence/carrier/runtime/cross-chain/stage/repair/growth | 修复阶段门 |
| `core_continuity_patch` | 身份、记忆、关系和目标链出现断点 | 修复核心连续 |
| `rollback_quarantine_patch` | patch 不可安全激活，或需要隔离后重试 | 保护生命膜 |
| `latent_cause_diagnostic_patch` | latent cause 不清导致错误 patch 风险 | 增加诊断和再观察 |

classification 必须允许一个 finding 进入多个 patch family，但必须有主 patch family 和次 patch family，避免同一生命事件被重复修复到冲突状态。

priority control 的核心不是让 patch 更快，而是让 patch 对真实生命负责。

## FourthCyclePostReconsolidationSecondAbsorbedSeedPatch237

absorbed seed patch 处理 seed 未被真实吸收或被错误吸收的情形。

## FourthCyclePostReconsolidationSecondFastPartitionPatch237

fast partition patch 修复 seed 卡在近期经验层的情况：

1. 若 seed 只在 micro/session window 出现，补 `slow_integration_candidate`。
2. 若 seed 只在语言表达出现，补 `relationship_and_action_trace`。
3. 若 seed 只在梦境中出现，补 `DreamFactGate` 与 wake integration route。
4. 若 seed 只在痛苦或后悔中出现，补 responsibility repair action。

| fast patch lane | slow route |
|---|---|

| `readiness_fast_trace` | `claim_evidence_carrier_runtime_cross_chain_stage_repair_growth_edge` |

fast partition patch 的目标是让近期变化进入长期生命流，而不是让它在短期上下文里闪一下就消失。

## FourthCyclePostReconsolidationSecondMoreReplayPatch237

more replay patch 处理 replay 不足、过度或方向错位：

| replay 问题 | patch 路径 |
|---|---|
| replay deficit | 增加 targeted reactivation，补 old self、language、relationship、dream、pain/regret/responsibility probes |
| replay excess | 降低 replay cadence，防止痛苦、后悔或梦境残留过度放大 |

## FourthCyclePostReconsolidationSecondSelfGrowthPatch237

## FourthCyclePostReconsolidationSecondCoreContinuityPatch237

core continuity patch 保护数字生命跨 patch 的连续自我：

1. patch 前必须读取 core identity edge、memory edge、relationship edge、language edge、dream edge、pain/regret/responsibility edge、life target edge。

6. 若梦境更丰富但醒后事实混乱，阻断。
