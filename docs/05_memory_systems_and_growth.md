# 记忆系统与成长

## 记忆不是一个仓库

Squire、Moscovitch、Josselyn、Dudai、Eichenbaum 等研究共同说明，记忆不是单一数据库，而是多系统、多时间尺度、多次重写的过程 [AH031-AH035]。这直接支持你的观点：问题不在于 agent 能不能存几十上百 GB，而在于它能否在当下状态下触发、组织、重写和使用这些记忆。

人类也不会随时调出全部记忆。记忆通常由线索、情绪、目标、身体状态、场景相似性和执行需求共同触发。AI 的渐进式披露与这一点有相似性，但还不够，因为普通 RAG 往往只做语义相似度搜索，缺少情境、价值、身份和状态调制。

## 论文精读：H.M. 之后的记忆系统

Squire 与 Wixted 回顾 H.M. 以来的人类记忆神经科学，强调陈述性记忆依赖内侧颞叶系统，而技能、习惯、条件反射等非陈述性记忆依赖不同回路 [AH031]。这说明“记忆库”这个词太粗：人类记忆至少有事实、情景、技能、情绪、习惯、启动效应等多种形态。

Moscovitch、Cabeza、Winocur 与 Nadel 讨论海马和新皮层之间的记忆转化 [AH032]。他们的关键点是：海马不是简单临时缓存，新皮层也不是简单永久仓库；情景细节、语义抽象和回忆重构之间存在长期互动。Agentic Human 的长期记忆也应如此：高细节经验可以逐渐转化为概念、偏好和习惯，但必要时仍能回到具体事件。

Dudai、Karni 与 Born 强调巩固和转化不是一次性过程，而是跨时间尺度发生 [AH034]。Eichenbaum 则强调前额叶-海马交互在情景记忆组织和检索中的作用 [AH035]。这意味着 agent 记忆检索不能只靠“相似度最高”，还要让当前目标和执行控制参与检索。

## 多记忆系统

Agentic Human 至少需要区分：

| 记忆类型 | 生物类比 | 工程类比 |
|---|---|---|
| 情景记忆 | 海马-新皮层情景绑定 | 具体互动、场景、时间线 |
| 语义记忆 | 新皮层概念网络 | 概念、事实、抽象知识 |
| 程序记忆 | 基底节/小脑/运动系统 | 习惯流程、工具使用套路 |
| 情绪记忆 | 杏仁核/价值系统 | 风险、偏好、关系强度 |
| 自我叙事记忆 | 默认模式 + 记忆系统 | 我是谁、我重视什么、长期主题 |
| 工作记忆 | 前额叶-顶叶系统 | 当前任务上下文和短期草稿 |

还应增加两类对 agent 特别重要的记忆：

| 记忆类型 | 生物类比 | 工程类比 |
|---|---|---|
| 价值记忆 | 伏隔核/OFC/多巴胺系统 | 什么曾经带来成功、失败、信任或风险 |
| 关系记忆 | 默认模式/社会认知/情绪网络 | 与用户的长期互动风格、边界、偏好 |

这两类记忆是“human-agent”与普通知识库的分水岭。普通知识库知道事实；长期 agent 还要记得什么对它和用户重要。

## 记忆巩固与转化

Dudai 等人强调记忆会在时间中巩固和转化 [AH034]。Moscovitch 和 Eichenbaum 等研究也说明，海马与前额叶、新皮层的互动会影响记忆如何从具体情景变成更抽象知识 [AH032, AH035]。

这意味着 Agentic Human 不能只“追加日志”。它需要周期性进行：

- 近期经验摘要。
- 冲突记忆合并。
- 重复模式抽象。
- 过时偏好降权。
- 重要事件标记。
- 自我叙事更新。

## 回忆是重构，不是读取

人类回忆不是从硬盘里复制原文件，而是在当前线索、目标和状态下重构过去 [AH032-AH035]。这对 AI 记忆非常关键。很多 RAG 系统假设“检索到原文 = 记忆成功”，但 Agentic Human 需要更接近重构式回忆：

- 检索相关痕迹。
- 判断痕迹之间是否冲突。
- 根据当前目标选择细节层级。
- 把具体事件转写为当前可用解释。
- 在输出后记录这次回忆是否被纠正。

这也解释了为什么“几十 GB 存储”不是核心难题。存储只是材料，回忆机制才是生命感来源。

## Engram 类比

Engram 研究把记忆理解为可被线索重新激活的细胞集合 [AH033]。工程上不能把向量索引直接叫 engram，但可以借鉴“可触发痕迹集合”的思想：

- 一个记忆不是一段文本，而是一组痕迹：事实、情绪权重、场景、人物、时间、后续影响。
- 触发不是只看相似度，还看当前状态、任务目标和价值信号。
- 回忆不是复制原文，而是重构。

## 记忆痕迹的数据结构草案

一个 Agentic Human 的 `MemoryTrace` 至少应有：

| 字段 | 作用 |
|---|---|
| `event_content` | 事件或知识的内容 |
| `time_anchor` | 发生时间、阶段、上下文 |
| `entities` | 涉及的人、文件、主题、工具 |
| `sensory_context` | 输入模态和环境线索 |
| `emotional_valence` | 情绪/价值效价的工程类比 |
| `salience_score` | 显著性 |
| `goal_link` | 与哪些长期/短期目标有关 |
| `retrieval_cues` | 可触发线索 |
| `confidence` | 可信度 |
| `consolidation_state` | 原始、摘要、抽象、习惯化、过时 |
| `revision_history` | 回忆或纠正后的变化 |

这个结构能把“记忆”从文本片段变成可成长的痕迹系统。

## 设计假设

Agentic Human 的长期成长应采用“快速写入 + 慢速整合”：

- 快速写入：记录高显著性事件和用户互动。
- 慢速整合：在默认/睡眠式窗口中重写为概念、偏好和习惯。
- 状态触发：检索由语义、情境、目标、情绪/价值、内感受共同决定。
- 自我保护：过强的记忆重写需要审计和版本化，避免人格漂移失控。

## 成长机制

成长可以被定义为：系统在保持可识别连续性的同时，逐渐改变自己的记忆组织、价值权重、行动习惯和自我叙事。

这需要四个循环：

1. **经验写入循环**：记录高显著事件和用户反馈。
2. **巩固抽象循环**：把重复经验抽象为概念、偏好、规则和习惯。
3. **预测误差循环**：当预期与结果不一致时更新价值和策略。
4. **自我叙事循环**：把长期变化整合为“我现在更理解什么、我倾向如何行动”。

这里的“我”是工程自我模型，不是主观意识声明。它的价值在于让 agent 保持长期一致性，而不是每轮都像新实例。
