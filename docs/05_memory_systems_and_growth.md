# 记忆系统与成长

## 记忆不是一个仓库

Squire、Moscovitch、Josselyn、Dudai、Eichenbaum 等研究共同说明，记忆不是单一数据库，而是多系统、多时间尺度、多次重写的过程 [AH031-AH035]。这直接支持你的观点：问题不在于 agent 能不能存几十上百 GB，而在于它能否在当下状态下触发、组织、重写和使用这些记忆。

人类也不会随时调出全部记忆。记忆通常由线索、情绪、目标、身体状态、场景相似性和执行需求共同触发。AI 的渐进式披露与这一点有相似性，但还不够，因为普通 RAG 往往只做语义相似度搜索，缺少情境、价值、身份和状态调制。

## 多记忆系统

Agentic Human 至少需要区分：

| 记忆类型 | 生物类比 | 工程类比 |
|---|---|---|
| 情景记忆 | 海马-新皮层情景绑定 | 具体互动、场景、时间线 |
| 语义记忆 | 新皮层概念网络 | 概念、事实、抽象知识 |
| 程序记忆 | 基底节/小脑/运动系统 | 习惯流程、工具使用套路 |
| 情绪记忆 | 杏仁核/价值系统 | 风险、偏好、关系强度 |
| 自我叙事记忆 | 默认模式 + 记忆系统 | 我是谁、我重视什么、长期主题 |
| 工作记忆 | 前额叶-顶叶系统 | 当前任务上下文和短期草稿 |

## 记忆巩固与转化

Dudai 等人强调记忆会在时间中巩固和转化 [AH034]。Moscovitch 和 Eichenbaum 等研究也说明，海马与前额叶、新皮层的互动会影响记忆如何从具体情景变成更抽象知识 [AH032, AH035]。

这意味着 Agentic Human 不能只“追加日志”。它需要周期性进行：

- 近期经验摘要。
- 冲突记忆合并。
- 重复模式抽象。
- 过时偏好降权。
- 重要事件标记。
- 自我叙事更新。

## Engram 类比

Engram 研究把记忆理解为可被线索重新激活的细胞集合 [AH033]。工程上不能把向量索引直接叫 engram，但可以借鉴“可触发痕迹集合”的思想：

- 一个记忆不是一段文本，而是一组痕迹：事实、情绪权重、场景、人物、时间、后续影响。
- 触发不是只看相似度，还看当前状态、任务目标和价值信号。
- 回忆不是复制原文，而是重构。

## 设计假设

Agentic Human 的长期成长应采用“快速写入 + 慢速整合”：

- 快速写入：记录高显著性事件和用户互动。
- 慢速整合：在默认/睡眠式窗口中重写为概念、偏好和习惯。
- 状态触发：检索由语义、情境、目标、情绪/价值、内感受共同决定。
- 自我保护：过强的记忆重写需要审计和版本化，避免人格漂移失控。
