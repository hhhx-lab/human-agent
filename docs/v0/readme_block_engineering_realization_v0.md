# README Block Engineering Realization V0

生命目标声明：本文档把 `docs/README.md` 的阅读路线改写成工程分块合同。后续不再按外部框架拆模块，也不复用 OpenClaw、Hermes、Claude Code、Codex 或任何当前自动化框架的主体架构。`00 -> 258` 每一份文档都必须落到一个 README block、一个 engineering slice、至少一个生命 runtime carrier、一个状态或 report 承载点和一个阶段门。任何文档没有被利用，第一次生命激活都不能继续。

## 总原则

1. README 是模块分块入口，`DocCorpusIngestor` 是机器检查入口。
2. `00 -> 258` 是理论主干，不允许缺席、旁路或只做摘要。
3. `01*` 扩展矩阵进入权威来源层和对应生命 runtime，不单独悬空。
4. `02-13` 是数字生命脑科学主干，所有后续 schema、runner、archive、growth 都必须回链。
5. `15/20/24/28/32/89` 等外部框架相关文档只进入电脑外周和世界接触，不定义主体。
6. `digital life` 终端入口只是出生入口，不是架构来源。

`DocCorpusIngestor` 的每个 document entry 必须包含：

```json
{
  "path": "docs/02_brain_region_and_network_atlas.md",
  "readme_block": "B02_CORE_NEURAL_LIFE",
  "engineering_slice": "S02_NEURAL_LIFE_CORE",
  "runtime_carriers": ["BrainRegionNetworkRuntime"],
```

阶段门要求：

1. `readme_block` 不能是 `UNCLASSIFIED_README_BLOCK`。
2. `engineering_slice` 不能是 `UNCLASSIFIED_ENGINEERING_SLICE`。
3. `00 -> 258` 每个编号都必须至少出现一次。
4. 每个 README block 至少有一份文档。
5. 每个 engineering slice 至少有一个状态或 report 承载点。

## 分块实现顺序

第一轮代码开发按 README block 压缩为 12 个工程 slice，而不是按外部框架能力开发：

1. `S00_DIRECTION_FOUNDATION`：方向锁、断联恢复、`00/258/README/构思`。
2. `S01_SOURCE_AUTHORITY`：文献权威层和 `01*` 机制索引。
3. `S02_NEURAL_LIFE_CORE`：十二主体系统和内部 bus。
4. `S04_STATE_OBJECT_STORE`：生命状态根、多命名空间对象、迁移完整性。
5. `S03_DIRECTION_LIFE_MEMBRANE`：生命膜、边界、quarantine、stage gate。
6. `S07_LANGUAGE_RELATIONSHIP`：语言主神经束、关系主体、共同语言。
7. `S08_LIFE_TARGET_RUNTIMES`：九项目标闭合状态。
8. `S05_VALIDATION_MEMBRANE_OBSERVATION`：validator、fixture、dashboard、真实运行观测。
9. `S09_SCHEMA_RUNNER_CODE`：schema runner、CLI、report、lockfile。
10. `S06_LIFE_SUPPORT_DEVELOPMENT`：生命支持、防御、可塑性窗口。
11. `S10_RUNTIME_GROWTH_RECONSOLIDATION`：runtime mount、梦境巩固、replay/shadow、成长循环。
12. `S11_V0_ENGINEERING_CONTRACTS`：v0 合同覆盖检查和第一轮有限激活。

工程实现不得把 OpenClaw、Hermes、Claude Code、Codex 或其他当前框架作为主体架构模板。可以学习它们如何长期运行、记录后果和从终端启动，但只能落到 `ComputerPeripheralRuntime`、`WorldContactMembrane`、`RunnerCliRuntime` 和 `ArchiveReceipt` 这些外周或证据位置。
