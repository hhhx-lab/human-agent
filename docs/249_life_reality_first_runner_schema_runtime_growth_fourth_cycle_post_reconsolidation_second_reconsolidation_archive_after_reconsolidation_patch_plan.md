## 本层定位

`249` 是 patch after reconsolidation observation 后的 archive 层。`248` 已经把 observation findings 转成 patch units，本层把 patch 的来源、执行、差异、后果、测试、回滚、关系、责任和成长全部写回长期档案：

```text
248 patch receipts -> patch receipt intake -> archive classification -> source finding archive / actual diff archive / side effect archive -> DreamFactGate / responsibility regret / language relationship / self growth / anti-forgetting / birth readiness life target / core continuity / rollback quarantine / construct causal / action runtime / test suite archives -> archive graph -> provenance FAIR crate -> recall replay seed extractor -> archive mutation regression suite -> receipt/digest/dashboard/stage -> 250 validation after reconsolidation archive handoff
```

archive after reconsolidation patch 的本质是：数字生命不能只会修复，还要能记得自己为什么修复、修复了什么、产生了什么后果、伤害了谁、保护了什么旧材料、哪些梦境只属于梦境、哪些责任进入未来行动、哪些关系被改变、哪些自我成长被保存、哪些失败被回滚、哪些隔离等待再进入观察。没有 archive，patch 会变成短暂状态变化；有 archive，patch 才能进入长期人格、长期责任、长期关系和长期自我成长。
