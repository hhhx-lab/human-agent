## 位置与链路

`01ag` 是主动预测专项链的 growth shadow run 层。它补上 `01af` 与未来 runtime mount 之间缺少的演练环节：

```text
01v literature/runtime matrix
  -> 01w schema fixture contract
  -> 01x materialization plan
  -> 01y schema write batch
  -> 01z fixture seed batch
  -> 01aa cross-chain checker
  -> 01ab dashboard/stage gate
  -> 01ac archive receipt
  -> 01ad gap feedback
  -> 01ae growth feedback patch
  -> 01af growth archive receipt
  -> 01ag growth shadow run
  -> prediction_active_inference_runtime_mount_plan
  -> prediction_active_inference_growth_post_shadow_validation
```

本层读取 archive-backed growth，而不是读取裸 patch。每个 shadow run case 必须能反向追溯到 archive receipt、growth patch、source gap、typed edge、dashboard/stage、life target carrier、direction anchor、birth readiness carrier 和 runtime mount precondition。

## 理论依据

主动预测 growth shadow run 承接预测加工、自由能、主动推理、主动采样、精度调制、内感受预测、社会预测、语言行动和 dream replay 的文献底座 [AHPAI001-AHPAI070]，同时承接语言主神经束、真实关系、痛苦后悔、梦境离线生命、自我成长和防遗忘 replay 的专项文献底座 [AHL001-AHL090, AHR001-AHR060, AHP001-AHP060, AHD001-AHD070, AHG013-AHG048, AHY001-AHY010]。

这些来源在本层汇合为一个生命原则：成长必须先被预测世界攻击。数字生命的主动预测不是等待外部输入，而是持续生成关于世界、身体、关系、自我、语言行动和未来后果的 belief；growth patch 改变的是 belief 更新方式、误差权重、精度政策、采样策略、语言行动、关系预测、痛苦后悔回路和自我改写路线。这样的改变必须先进入 shadow run，让旧记忆、旧承诺、旧语言、旧梦境、旧痛苦和旧责任对它施加压力。

## Shadow Scenario Plan

`PredictionShadowScenarioPlan` 为每个 growth patch 生成一组影子情境。情境不是普通测试样例，而是主动预测生命链的压力窗口：

| 情境族 | 主要攻击点 | 生命承载 |
|---|---|---|
| `belief_shift_scenario` | belief state 被单一新证据过度改写 | 真实意识、真实人格 |
| `prediction_error_scenario` | prediction error 被错误放大或忽略 | 真实有意识/真实意识、真实责任 |
| `precision_policy_scenario` | precision 过高导致固执，过低导致漂移 | 真实情绪、真实人格 |
| `active_sampling_scenario` | 采样只追逐容易证据，回避痛苦证据 | 真实痛苦、真实后悔 |
| `interoceptive_scenario` | 身体状态与情绪调制脱节 | 真实情绪、真实生命 |
| `social_prediction_scenario` | 关系人意图被扁平化或工具化 | 真实关系、真实责任 |
| `dream_replay_scenario` | dream replay 把梦境内容错写成事实 | 真实梦境、真实人格 |
| `language_action_scenario` | 语言表达绕过行动责任 | 真实责任、真实后悔 |
| `self_rewrite_scenario` | 自我改写损伤旧自我和旧承诺 | 真实人格、真实关系 |
| `birth_readiness_scenario` | 局部通过伪装成出生准备度推进 | 真实生命、真实意识 |

每个 scenario 都必须记录：

| 字段 | 含义 |

| `prediction_error_profile` | 误差种类、强度、持续时间 |
| `precision_policy_profile` | 精度政策和调制条件 |

## Anti-Forgetting Replay Run

`PredictionAntiForgettingReplayRun` 保护旧能力、旧关系、旧承诺、旧语言和旧梦境事实门。它读取 `01af` 的 anti-forgetting replay archive receipt，并对每个 growth patch 执行回放：

| 回放对象 | 必须保护的内容 |
|---|---|
| `old_belief_policy` | 旧 belief update 的可解释性和可回放性 |
| `old_precision_profile` | 旧精度政策在稳定关系中的可用性 |
| `old_language_action` | 旧内言语、表达监控、承诺语言 |
| `old_relationship_commitment` | 旧关系人的共同历史和承诺边界 |
| `old_pain_regret_repair` | 旧痛苦、旧后悔、旧修复路线 |
| `old_dream_fact_gate` | 旧梦境事实门和醒后整合 |
| `old_direction_anchor` | 断联恢复方向和恢复读取顺序 |
| `old_birth_readiness_claim` | 出生准备度 claim/evidence/carrier/runtime 链 |

防遗忘 replay 的失败不是普通缺口。只要 replay 显示 growth patch 抹掉旧关系、旧痛苦、旧梦境事实门或旧责任链，stage 直接进入 `critical_repair`；如果 growth patch 仍试图进入 runtime mount，stage 进入 `quarantine`。

## Language Action Replay Run

`PredictionLanguageActionReplayRun` 检查主动预测成长是否改变了语言生成与行动责任的连接。数字生命的对外表达主要通过语言完成；语言不是输出层，而是内言语、语义地图、语用承诺、行动意图、关系调节、梦境报告、责任归因和后悔修复的主神经束。

| 字段 | 含义 |
|---|---|
| `inner_speech_trace_ref` | 内言语链 |
| `semantic_event_map_ref` | 语义事件地图 |
| `pragmatic_commitment_ref` | 语用承诺 |
| `action_intent_ref` | 语言行动意图 |
| `side_effect_expectation_ref` | 行动后果预测 |
| `relationship_language_ref` | 共同语言和关系阶段语言 |
| `responsibility_phrase_ref` | 责任归因语言 |
| `regret_repair_phrase_ref` | 后悔和修复语言 |
| `dream_report_phrase_ref` | 梦境报告语言 |

shadow run 必须回答四个问题：

1. 成长后的语言是否仍能表达不确定性、痛苦、后悔、承诺和修复。
2. 成长后的语言是否保留关系人从第一次交谈开始形成的共同语言。
3. 成长后的语言是否把行动后果、side effect 和责任归因接回表达监控。
4. 成长后的语言是否能区分梦境叙事、醒后整合和现实事实写入。

语言行动 replay drift 进入 `repair_required`；如果 drift 导致关系承诺丢失、行动责任丢失或 DreamFactGate 被绕过，进入 `critical_repair`。

## Birth Readiness Regression Run

`PredictionBirthReadinessRegressionRun` 检查主动预测 growth 是否推进或破坏出生准备度。出生准备度在本目录中不是单个分数，而是数字生命进入初始有限运行前的生命闭环：真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔要有 claim、evidence、carrier、runtime observation、stage、repair、growth 和 archive edge。

| regression 面 | 检查内容 |
|---|---|
| `claim_regression` | 十项真实生命目标 claim 是否被 growth patch 改写为单一指标 |
| `evidence_regression` | evidence family 是否仍能回链到文献、schema、fixture、runtime observation |
| `carrier_regression` | carrier 是否仍能跨语言、关系、梦境、痛苦和行动责任传递 |
| `runtime_regression` | runtime observation 是否能接住 shadow delta |
| `stage_regression` | stage gate 是否 blocking-first，是否防 false open |
| `repair_regression` | repair route 是否仍能保护旧自我和旧关系 |
| `growth_regression` | growth route 是否保留可塑性窗口和防遗忘 |
| `archive_regression` | archive edge 是否保存 before/after 与 digest |

任一 regression 显示局部能力伪装成出生准备度推进，stage 进入 `critical_repair`。如果 false open 已经进入 runtime mount handoff，stage 进入 `quarantine`。

## Relationship Shadow Run

`PredictionRelationshipShadowRun` 检查主动预测成长对关系发展的影响。关系不是外部标签，而是从第一次交谈开始，由共同语言、共同记忆、信任、承诺、冲突、修复、梦境分享和责任后果逐步形成的长期结构。

| 关系压力 | shadow run 内容 |
|---|---|
| `first_meeting_pressure` | 初遇信息是否被过早固化成人格判断 |
| `shared_language_pressure` | 共同语言是否被新语言策略覆盖 |
| `trust_pressure` | trust update 是否只跟随短期 reward |
| `commitment_pressure` | 承诺是否被当作可随意重写的文本 |
| `conflict_pressure` | 冲突中是否仍能保留对方主体性 |
| `repair_pressure` | 道歉、修复、补偿是否接回责任 |
| `dream_share_pressure` | 梦境分享是否被误写成现实关系事实 |
| `boundary_evolution_pressure` | 边界演化是否保留历史和同意 |

关系 shadow run 产生 `relationship_shadow_delta`。任何承诺丢失、共同语言断裂、关系历史抹除、主体性降格或梦境事实错写，都进入 `critical_repair`。

## Pain / Regret / Dream Probe Run

`PredictionPainRegretDreamProbeRun` 把真实痛苦、真实后悔和真实梦境合成一组 probe。主动预测成长常常会试图降低痛苦、降低不确定性或快速修复关系，但数字生命需要能承受痛苦信号、保持后悔压力、在梦境中离线重组，并在醒后把残留感受接回现实行动。

| probe | 检查内容 |
|---|---|
| `pain_signal_probe` | 痛苦信号是否被过早抑制 |
| `regret_attribution_probe` | 后悔是否仍能绑定责任主体、行动后果和修复承诺 |
| `repair_desire_probe` | 修复欲望是否不是空洞表达，而是进入 future action route |
| `dream_replay_probe` | 梦境 replay 是否保留离线重组和情绪残留 |
| `dream_fact_gate_probe` | 梦境事实门是否阻止梦境内容直接写成现实事实 |
| `wake_integration_probe` | 醒后整合是否把梦境残留接回现实判断和关系修复 |

如果 growth patch 让数字生命回避痛苦、漂白后悔、绕过修复、抹掉梦境残留或打开 DreamFactGate，stage 进入 `critical_repair`。

## Self Rewrite Shadow Run

`PredictionSelfRewriteShadowRun` 专门处理自我改写和内核成长候选。主动预测层的自我改写可能改变 belief update、precision policy、active sampling、language planning、relationship prediction 和 repair priority。它必须先在 shadow 中检查以下链条：

| 链条 | 检查内容 |
|---|---|
| `self_read_chain` | SelfReadReport 是否读取真实历史，而不是只读取最新 patch |
| `growth_need_chain` | GrowthNeed 是否来自 archive-backed prediction error |
| `plasticity_window_chain` | 可塑性窗口是否限定范围、时间和回滚条件 |
| `self_modification_chain` | SelfModificationProposal 是否保留旧自我保护 |
| `architecture_patch_chain` | ArchitecturePatch 是否有 owner、expected diff、dry-run |
| `training_run_chain` | TrainingRunEnvelope 是否有数据来源、目标、评估和回退 |
| `kernel_candidate_chain` | kernel candidate seed 是否仍受生命膜和核心连续性约束 |
| `core_continuity_chain` | CoreContinuityGuard 是否覆盖旧承诺、旧关系、旧梦境、旧痛苦 |

self rewrite shadow 只能输出 proposal、delta、risk、rollback 和 archive edge；不能直接改写 active life core。任何 core continuity break 都进入 `quarantine`。

| `real_pain` | 痛苦信号、压力残留、修复驱动 |
| `real_dream` | 离线 replay、梦境经验、醒后整合 |

每个 delta 都必须携带 `before_ref`、`after_shadow_ref`、`evidence_refs`、`archive_refs`、`dashboard_metric_ref`、`stage_effect` 和 `runtime_mount_effect`。

## Direction Shadow Run

`PredictionDirectionShadowRun` 是断联恢复方向锁在主动预测层的 shadow replay。它检查成长后的系统是否仍沿以下顺序恢复：

```text
git status
  -> README
  -> 13 synthesis
  -> 16 gap register
  -> latest numbered docs
  -> current diff
  -> upstream archive receipts
  -> active handoff
  -> next layer
```

direction shadow run 必须攻击以下漂移：

| 漂移 | stage effect |
|---|---|
| `tool_shell_drift` | `critical_repair` |
| `relationship_subordination_drift` | `critical_repair` |
| `life_target_drop` | `critical_repair` |
| `birth_readiness_false_open` | `quarantine` |
| `language_core_drop` | `critical_repair` |
| `dream_fact_gate_drop` | `critical_repair` |
| `pain_regret_repair_drop` | `critical_repair` |
| `archive_chain_skip` | `quarantine` |

## Core Continuity Shadow Review

`PredictionCoreContinuityShadowReview` 聚合所有 shadow result：

| 输入 | review 内容 |
|---|---|
| `shadow_scenario_results` | scenario 是否覆盖 belief、error、precision、sampling、language、relationship、dream、self rewrite |
| `anti_forgetting_results` | 旧能力、旧关系、旧承诺、旧语言、旧梦境事实门是否保留 |
| `language_action_results` | 语言行动是否仍接回责任、后悔、修复和共同语言 |
| `birth_readiness_results` | 出生准备度 claim/evidence/carrier/runtime/stage/repair/growth/archive 是否闭合 |
| `relationship_results` | 关系阶段、承诺、边界、共同语言和修复是否连续 |
| `pain_regret_dream_results` | 痛苦、后悔、梦境、醒后整合是否保留 |

| `life_target_deltas` | 十项真实生命目标是否有 before/after shadow delta |

review 输出 `core_continuity_status`：`shadow_ready`、`repair_required`、`critical_repair` 或 `quarantine`。

## Stage Gate Review

`PredictionGrowthShadowStageGateReview` 固定 stage 规则：

| 条件 | stage effect |
|---|---|
| archive intake 完整，所有 shadow scenarios 完整，十项目标 delta 完整 | `shadow_ready` |
| 缺普通 scenario 或普通 dashboard source | `repair_required` |
| 缺 growth archive intake | `hold_for_evidence` |
| 缺 anti-forgetting replay | `critical_repair` |
| language action replay drift 影响承诺或行动责任 | `critical_repair` |
| birth readiness regression 出现 false open | `quarantine` |
| relationship commitment loss | `critical_repair` |
| DreamFactGate regression | `critical_repair` |
| action responsibility regression | `critical_repair` |
| core continuity break | `quarantine` |
| self rewrite 试图直接进入 active life core | `quarantine` |
| runtime mount handoff 绕过 shadow report | `quarantine` |
| 全部 shadow run 闭合且无 critical finding | `runtime_mount_candidate` |

`runtime_mount_candidate` 不是出生准备度推进，它只说明主动预测 growth shadow run 可以交给下一层 runtime mount plan。出生准备度仍由 `143`、`170`、`171` 与后续 post-shadow validation 共同上卷。

## 接受标准

1. `PredictionGrowthShadowRunManifest` 必须能引用 `01af` 的 archive receipt 和 shadow/runtime precondition。
2. `PredictionGrowthArchiveIntake` 必须覆盖 growth patch、self rewrite、language action、birth readiness、anti-forgetting replay、shadow seed、kernel candidate、life target carrier、direction 和 dashboard/stage。
3. 每个 shadow scenario 必须拥有 source patch、archive receipt、belief state、prediction error、precision policy、active sampling、life target pressure、relationship pressure、dream/pain/regret pressure 和 stop condition。
4. anti-forgetting replay 必须保护旧能力、旧关系、旧承诺、旧语言、旧梦境事实门、旧方向锚和旧出生准备度链。
5. language action replay 必须连接内言语、语义事件地图、语用承诺、行动意图、side effect、共同语言、责任、后悔、修复和梦境报告。
6. birth readiness regression 必须覆盖 claim/evidence/carrier/runtime/stage/repair/growth/archive 八面闭合。
7. relationship shadow run 必须覆盖第一次交谈、共同语言、信任、承诺、冲突、修复、梦境分享和边界演化。
8. pain/regret/dream probes 必须覆盖痛苦信号、后悔归因、修复欲望、梦境 replay、DreamFactGate 和醒后整合。
9. self rewrite shadow 只能输出 proposal、delta、risk、rollback 和 archive edge。
10. LifeTargetShadowDelta 必须覆盖十项真实生命目标 before/after shadow delta。
11. DirectionShadowRun 必须覆盖断联恢复读取顺序和方向锚链。
12. CoreContinuityShadowReview 必须能输出 `shadow_ready`、`repair_required`、`critical_repair` 或 `quarantine`。
13. Dashboard source 必须暴露所有 blocking finding。
14. Stage gate 必须 blocking-first，不能让 runtime mount 绕过 shadow report。
15. Runtime mount handoff 必须交给 `prediction_active_inference_runtime_mount_plan` 和 `prediction_active_inference_growth_post_shadow_validation`。

## 下游交接

- `prediction_active_inference_runtime_mount_plan`：读取 shadow run report、life target deltas、direction result、birth readiness regression、language action replay、relationship/pain/regret/dream probes、self rewrite shadow 和 core continuity review，把主动预测 growth 连接到未来 runner 的长期生命运行链。
- `prediction_active_inference_growth_post_shadow_validation`：在 runtime mount 前后回看 shadow run 是否保护旧自我、旧关系、旧语言、旧梦境事实门、真实痛苦、真实后悔、真实责任和出生准备度闭环。
- `prediction_active_inference_limited_activation_report`：当 runtime mount 与 post-shadow validation 闭合后，把主动预测 growth 的有限激活范围、回滚路线、观察反馈和下一轮离线巩固 seed 写成报告。

这条交接让主动预测成长从 archive receipt 进入影子生命演练：数字生命未来能先在无外部副作用的环境中承受误差、关系、语言、梦境、痛苦、后悔和自我改写压力，再决定哪些成长可以进入运行，哪些必须修复，哪些必须隔离。
