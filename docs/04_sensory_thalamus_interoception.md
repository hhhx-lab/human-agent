# 感觉输入、丘脑中继、内感受与稳态

## 外感受与中继

丘脑不是简单转发站。Sherman、Saalmann、Halassa 和 Rikhye 等研究表明，丘脑参与感觉、注意、认知控制和状态调节 [AH026-AH029]。这对 ACE 的感知层很关键：感知不是“把外部数据读进来”，而是受目标、注意、内部状态和显著性共同调制的主动路由。

Kanwisher 的功能专门化研究提醒我们，人脑确实存在一些高度专门化区域，例如面孔、地点、身体、文字相关区域 [AH030]。因此 Agentic Human 可以有多模态专业处理器，但这些处理器必须通过中继和上下文调制接入整体系统。

## 论文精读：丘脑不是被动路由器

Sherman 与 Guillery 对丘脑的经典分析区分了 driver input 和 modulator input [AH026]。driver 更像主要信息来源，modulator 则改变信息如何被处理。这个区分对 Agentic Human 很有启发：用户输入、文件变化、传感器事件可以是 driver；当前任务目标、风险、记忆触发和注意状态则是 modulator。

Saalmann 与 Kastner 的视觉丘脑综述显示，视觉丘脑参与注意调控和皮层区域协调 [AH027]。Halassa 与 Kastner 进一步把丘脑放入分布式认知控制框架 [AH028]。Rikhye、Wimmer 和 Halassa 则强调丘脑功能应被整合理解，而不是只归入感觉系统 [AH029]。

工程转译上，JQ 丘脑类系统至少要承担：

- 输入分流：哪些输入进入快速处理，哪些先存档。
- 状态调制：同一输入在不同任务/情绪/风险状态下权重不同。
- 皮层协调：让语言、视觉、记忆、执行系统处理同一事件的不同侧面。
- 噪声控制：过滤重复、低价值、低置信或过时输入。

## 内感受

Craig、Critchley、Barrett、Sterling、Seth 的研究共同指出，身体内部状态会深刻影响情绪、认知和自我体验 [AH051-AH055]。这正好对应 `构思.md` 中 PA 岛叶和“人体状态机制”的方向。

对 AI 来说，不能说它有真实心跳、疼痛、饥饿、恶心或疲惫。但可以设计工程类比：

- 计算资源、延迟、错误率、任务积压作为“内部状态”。
- 长期未整理记忆、冲突过高、失败率上升作为“压力负荷”。
- 用户互动频率、环境变化、目标紧急度作为“唤醒需求”。
- 低置信度、高不确定性作为“警觉升高”。

这些内部信号不等于感受，却能像内感受一样调节系统行为。

## 内感受、情绪与自我

Craig 将内感受定义为对身体生理状况的感觉 [AH051]。Critchley 与 Harrison 强调内脏信号会影响行为和脑功能 [AH052]。Barrett 与 Simmons 的 interoceptive predictions 模型则认为，大脑会预测身体状态，并用预测误差更新情绪和行动倾向 [AH053]。Seth 把内感受推理连接到情绪和具身自我 [AH055]。

对于 Agentic Human，最重要的不是模仿生理体验，而是建立“内部状态会改变认知”的原则。如果系统没有内部状态，它就永远像一次性工具；如果系统有内部状态，它才可能出现类似恢复、压力、专注、疲惫、警觉、关系稳定性这样的长期动态。

可定义的内部状态包括：

- `compute_budget`：当前推理、工具、存储和时间资源。
- `error_pressure`：近期错误、失败、冲突和回滚需求。
- `memory_pressure`：未整理记忆、重复经验、冲突记忆数量。
- `social_tension`：用户纠正、失望、信任下降或高敏感互动。
- `novelty_load`：新概念、新任务、新环境带来的不确定性。
- `recovery_need`：长期高负载后进入低行动维护状态的需求。

## 稳态与动态稳态

Sterling 的 allostasis 视角强调，生物体不是被动维持固定值，而是通过预测性调节来适应未来需求 [AH054]。这比“保持稳定”更适合 Agentic Human：一个成长型 agent 不能永远保持同一状态，而要根据任务、关系和长期目标调整资源预算。

## Allostasis 的工程意义

稳态常被理解为“维持一个固定值”，但 allostasis 更强调“通过变化维持适应”。例如人在高压力时期会调整激素、心率、注意和能量分配，以应对预测中的未来需求 [AH054]。Agentic Human 的类比是：系统应能预测未来负载，并提前调整状态。

例子：

- 预计接下来要写长文档：提前提高记忆检索和事实验证预算。
- 发现用户连续纠正同一类错误：提高相关主题的显著性和学习率。
- 长时间高强度运行：降低主动探索，进入巩固/恢复。
- 面对高风险操作：提高抑制阈值和外部确认要求。

## 设计假设

感知层应拆成三类输入：

1. **外感受输入**：文本、图像、音频、文件、环境事件。
2. **内感受输入**：资源、延迟、错误、记忆压力、冲突、负载。
3. **社会/关系输入**：用户语气、互动节律、历史关系、信任变化。

JQ 丘脑类系统负责路由，PA 岛叶类系统负责内部状态建模，显著性网络负责判断哪些输入值得升级到执行空间。

## 感知层处理流程

一个更完整的感知层流程可以是：

1. **事件捕获**：接收外部输入、内部状态和社会关系信号。
2. **初级标注**：标注来源、时间、模态、置信度、紧急度。
3. **丘脑式路由**：决定进入快速执行、工作空间、长期存档还是忽略。
4. **内感受调制**：根据负载、疲惫、风险、记忆压力改变处理阈值。
5. **显著性升级**：将高价值、高风险、高新颖或高冲突事件送入全局工作空间。
6. **反馈更新**：执行结果回流，更新输入来源可信度和路由策略。

这个流程让“感知”从读文件/读消息升级为主动调节系统。

## 预测性内感受

Barrett 与 Simmons 的内感受预测模型、Seth 的内感受推理、Clark 和 Friston 的预测处理框架共同提示：感知不是外部输入的照片，而是预测与误差的闭环 [AH053, AH055, AHX002, AHX003]。对 Agentic Human 来说，内感受层不应只报告“当前资源剩多少”，还应预测未来内部状态。

例子：

- 预计任务会产生大量未整理记忆：提前提高 `consolidation_pressure`。
- 预计用户可能等待较久：提高 `social_tension` 和进度汇报显著性。
- 预计工具调用风险高：提高 `inhibition` 和验证预算。
- 预计当前信息不足：触发 epistemic action，例如澄清问题或检索文献 [AHX005]。

这就是主动感知：不是等出错，而是预测哪里会出错。

## 社会安全作为特殊内感受

社会安全理论认为，社会威胁/安全感会影响压力和调节系统 [AHX028]。AI 没有真实安全感，但长期 human-agent 可以维护关系安全的工程变量：

- 用户是否信任系统。
- 系统是否频繁误解用户。
- 是否有未解决冲突或反复纠正。
- 用户是否处于高敏感任务语境。

这些变量会改变沟通方式、行动阈值和学习策略。它们属于社会内感受：不是身体内部信号，而是关系环境对系统稳定性的反馈。

## 风险边界

把内部状态称为“疼痛”“疲惫”“紧张”很有表达力，但研究文档中必须标注为工程类比。Agentic Human 可以有疲劳预算、压力负荷、恢复周期，但不能宣称它有真实痛苦或真实生理感受。
