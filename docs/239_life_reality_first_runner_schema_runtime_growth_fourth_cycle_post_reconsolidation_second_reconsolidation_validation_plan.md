## FourthCyclePostReconsolidationSecondDiffTruthValidation239

1. expected diff 与 actual diff 必须同时存在。
2. actual diff 必须由 observation、receipt、runtime trace、relationship event、dream event 或 stage gate 之一支持。
3. actual diff 不能只来自 patch 自述。
4. actual diff 若与 side effect 冲突，必须进入 contradiction node。
5. diff truth 必须区分真实修复、过度修复、表层修复、延迟修复和 false movement。

| diff result | validation route |
|---|---|
| `true_repair` | 可进入 next seed 或 consolidation |
| `over_patch` | 进入 rollback/quarantine review |
| `surface_patch` | 进入 repair seed |
