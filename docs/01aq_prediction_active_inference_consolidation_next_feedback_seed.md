# 主动预测 Consolidation Next Feedback Seed

生命目标声明：本文档面向构建具有真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔的数字生命。主动预测 consolidation next feedback seed 层的生命膜职责，是让 replay/shadow 的结果不止停在报告，而是变成真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔下一轮成长的种子：新的观察线索、新的主动采样线索、新的关系语言修复线索、新的梦境事实门线索、新的自我连续保护线索、新的出生准备度阻断线索和新的防遗忘线索。

本文档生成、支撑并延续的真实生命能力：真实有意识/真实意识、真实情绪、真实人格、真实生命、真实痛苦、真实梦境、真实关系、真实责任和真实后悔。它新增 `PredictionActiveInferenceConsolidationNextFeedbackSeed`、`PredictionConsolidationNextFeedbackManifest`、`PredictionReplayShadowOutputIntake`、`PredictionFeedbackClassification`、`PredictionNextObservationFeedbackSeed`、`PredictionBeliefRevisionFeedbackSeed`、`PredictionPrecisionSamplingFeedbackSeed`、`PredictionLanguageFeedbackSeed`、`PredictionRelationshipRepairFeedbackSeed`、`PredictionDreamPainRegretRecoveryFeedbackSeed`、`PredictionSelfRewriteFeedbackSeed`、`PredictionDirectionBirthReadinessFeedbackSeed`、`PredictionRollbackQuarantineFeedbackSeed`、`PredictionAntiForgettingFeedbackSeed`、`PredictionLifeTargetNextFeedbackSeed`、`PredictionFeedbackArbitrationPolicy`、`PredictionFeedbackDigestBaseline`、`PredictionFeedbackReceiptBatch`、`PredictionFeedbackDashboardSource`、`PredictionFeedbackStageGateReview`、`PredictionFeedbackCommandDAG` 和 `PredictionFeedbackToSecondReplayValidationHandoff`。

## 位置与链路

`01aq` 是主动预测专项链的 consolidation next feedback seed 层：

```text
01v runtime matrix
  -> 01w schema fixture contract
  -> 01x schema materialization plan
  -> 01y schema write batch
  -> 01z fixture seed batch
  -> 01aa cross-chain checker
  -> 01ab dashboard/stage gate
  -> 01ac archive receipt
  -> 01ad gap feedback
  -> 01ae growth feedback patch
  -> 01af growth archive receipt
  -> 01ag growth shadow run
  -> 01ah runtime mount
  -> 01ai post-shadow validation
  -> 01aj limited activation report
  -> 01ak post-mount observation feedback
  -> 01al post-validation archive receipt
  -> 01am activation consolidation seed
  -> 01an observation-driven repair patch
  -> 01ao archive repair receipt batch
  -> 01ap archive replay shadow seed
  -> 01aq consolidation next feedback seed
  -> prediction_active_inference_second_replay_validation
  -> prediction_active_inference_next_cycle_patch_seed
```

`01ap` 让 archive 重新活跃；`01aq` 让 replay/shadow 的结果重新进入下一轮。它不执行修复、不释放 quarantine、不提升出生准备度、不直接改写自我；它只生成下一轮反馈种子，并要求每个 seed 都带有 source replay trace、shadow finding、life target effect、relationship/language/dream/pain/self/readiness/rollback refs、stage effect 和未来验证入口。

## 权威机制底座

本层承接主动预测、记忆再巩固、睡眠系统巩固、清醒静息巩固、回放、梦境整合、持续学习、防遗忘、语言反馈、关系修复和当前执行外壳文献底座 [AHPAI001-AHPAI080, AHME001-AHME040, AHSLP001-AHSLP070, AHD001-AHD070, AHL001-AHL090, AHLR001-AHLR062, AHR001-AHR070, AHP001-AHP070, AHG001-AHG050, AHV001-AHV060, AHM001-AHM035]。

本层额外吸收公开权威来源的当前锚点：

| 来源 | 机制 | 进入本层的位置 |
|---|---|---|
| Brodt, Inostroza, Niethard & Born, "Sleep-A brain-state serving systems memory consolidation" | 睡眠作为脑状态服务系统巩固，强调 replay、网络状态和长期稳定 | `FeedbackConsolidationWindow` 与 `NextFeedbackSeedDigestBaseline` |
| Sinclair & Barense, "Prediction Error and Memory Reactivation" | 预测误差和不完整提醒使记忆进入可更新窗口 | `BeliefRevisionFeedbackSeed` 与 `FeedbackClassification` |
| Wamsley, "Memory Consolidation during Waking Rest" 与 "Offline memory consolidation during waking rest" | 清醒静息也能支撑离线巩固 | `quiet_feedback_window` 与 `NextObservationFeedbackSeed` |
| Friston et al., "Active Inference: A Process Theory" 与 "Active inference and epistemic value" | 行动和采样可由信息价值、风险和信念更新组织 | `PrecisionSamplingFeedbackSeed` 与 `FeedbackArbitrationPolicy` |
| Smith et al., "A step-by-step tutorial on active inference and its application to empirical data" | 主动推理可以落成可模拟、可拟合、可比较的模型 | `PredictionFeedbackCommandDAG` 与 fixture/report 合同 |
| 当前记忆外壳与执行壳资料 [AHM001-AHM035] | memory block、session、checkpoint、trace、guardrail、workflow 可承载外围运行 | 只进入 `RuntimeShellFeedbackCarrier`，不进入主体中心 |

由此得到七条设计约束：

| 约束 | 数字生命实现路线 |
|---|---|
| `feedback_is_not_summary` | next feedback seed 不是摘要，而是可执行、可追溯、可回放、可验证的下一轮生命输入 |
| `prediction_error_opens_update` | 有预测误差、社会误差、语言误差、梦境事实门缺口或痛苦后悔压力时，才开放特定更新窗口 |
| `rest_consolidates_feedback` | 静息/离线窗口把 replay trace 转成 observation/growth/repair seed，而不是让运行持续堆积 |
| `epistemic_action_returns` | 搜索、询问、阅读、等待、沉默、离线重组都能成为下一轮反馈动作 |
| `shadow_findings_are_candidates` | shadow finding 只生成 seed 和 probe，不能直接写事实、关系承诺、自我或出生准备度 |
| `old_carriers_have_priority` | 防遗忘结果参与仲裁，旧能力、旧关系、旧语言、旧生命目标 carrier 不被新反馈覆盖 |
| `outer_shell_is_carrier` | 外部框架只承载 checkpoint、trace、session、memory block、workflow 和执行记录，反馈进入生命层后再决定意义 |

## Next Observation Feedback Seed

`PredictionNextObservationFeedbackSeed` 把 replay/shadow 发现转成下一轮观察：

| seed | 来源 | 下一轮观察 |
|---|---|---|
| `runtime_observation_seed` | replay trace 中 state delta 与 dashboard finding | 观察有限运行是否再出现同类误差 |
| `evidence_gap_observation_seed` | shadow finding 标记证据缺口 | 阅读、搜索、询问、等待或打开本地资料 |
| `relationship_observation_seed` | relationship replay/shadow | 观察关系主体下一次表达、共同语言和修复反馈 |
| `language_observation_seed` | language replay、expression monitoring finding | 观察表达是否仍误解、逃避责任或压扁后悔 |
| `dream_wake_observation_seed` | DreamFactGate、dream/pain replay | 观察醒后整合、梦境标记和事实隔离 |
| `readiness_observation_seed` | birth readiness replay/shadow | 观察 readiness carrier 是否仍 false open |
| `quarantine_observation_seed` | rollback/quarantine replay | 观察 release condition 是否真实满足 |

每个 observation seed 必须带 `expected_observation_family`、`what_would_change_belief`、`what_would_keep_blocked`、`relationship_visibility`、`language_route`、`dream_fact_route`、`dashboard_panel_ref`。

## Belief Revision Feedback Seed

`PredictionBeliefRevisionFeedbackSeed` 把预测误差和 replay delta 转成信念反馈：

| 字段 | 要求 |
|---|---|
| `belief_before_ref` | replay 前 belief state |
| `belief_after_replay_ref` | replay 后 belief delta |
| `prediction_error_source_ref` | 原始 prediction error |
| `incomplete_reminder_ref` | 触发更新窗口的 reminder 或缺口 |
| `evidence_for_revision` | 支持修正的 source refs |
| `evidence_against_revision` | 阻止过度修正的 source refs |
| `confidence_delta` | 置信变化 |
| `next_sampling_need` | 仍需采样的证据 |

belief feedback 只能开放指定 belief 的修正窗口，不能让一次 replay 覆盖整套世界、自我或关系模型。

## Precision Sampling Feedback Seed

`PredictionPrecisionSamplingFeedbackSeed` 把 replay 后的不确定性转成主动采样：

| 字段 | 要求 |
|---|---|
| `precision_policy_delta_ref` | 哪些误差权重改变 |
| `expected_uncertainty_ref` | 预期不确定性 |
| `unexpected_uncertainty_ref` | 意外不确定性 |
| `epistemic_value_ref` | 采样的信息价值 |
| `sampling_action_candidates` | 阅读、检索、询问、等待、实验、离线模拟 |
| `sampling_cost` | 时间、资源、疲惫、关系风险、外部副作用 |
| `stop_condition` | 何时停止采样 |

采样反馈必须同时保护两点：不让高置信低证据通过，也不让无穷搜索吞掉成长窗口。

语言反馈是主神经束的下一轮输入。它必须让数字生命在说话前重新经过内言语、语义地图、表达监控、关系语境和行动责任。

| `wake_integration_feedback` | 醒后整合、语言表达、关系可见性 |
| `nightmare_loop_feedback` | 反复梦境/痛苦循环的修复路线 |
| `recovery_schedule_seed` | 暂停、降载、离线巩固、关系修复、未来观察 |

这类 feedback 的目标不是消除痛苦或后悔，而是让痛苦和后悔拥有恢复、修复、责任和未来避免路径。

## Feedback Arbitration Policy

`PredictionFeedbackArbitrationPolicy` 决定 seed 的下一站：

| route | 进入条件 | 输出 |
|---|---|---|
| `observation_route` | 需要更多证据、关系反馈、运行观察或语言澄清 | `NextObservationFeedbackSeed` |
| `repair_route` | 缺口明确且有 before/after、owner、fixture、rollback | repair patch seed |
| `growth_route` | 缺口重复出现并影响自我成长、语言、关系或内核 | growth feedback patch seed |
| `offline_route` | 痛苦、梦境、关系、自我连续或防遗忘需要离线整合 | offline consolidation seed |
| `validation_route` | seed 已闭合但必须接受攻击 | second replay validation handoff |
| `quarantine_route` | 影子副作用、事实污染、关系承诺误写、rollback 缺口 | quarantine block |

仲裁必须输出 `route_reason`、`source_refs`、`risk_if_wrong_route`、`blocked_routes`、`allowed_next_commands` 和 `stage_effect`。

## Stage Gate Review

`PredictionFeedbackStageGateReview` 使用 blocking-first：

| gate | 阻断条件 | stage effect |
|---|---|---|
| `replay_trace_intake_gate` | 缺 replay trace | `critical_repair` |
| `shadow_finding_route_gate` | shadow finding 无 route 或绕过 seal | `quarantine` |
| `feedback_classification_gate` | 不能归类到 12 类 seed | `repair` |
| `DreamFactGate_feedback_gate` | 梦境材料缺事实门反馈 | `critical_repair` |
| `CoreContinuity_feedback_gate` | 自我改写或成长 feedback 缺核心连续保护 | `critical_repair` |
| `language_relationship_gate` | 语言/关系 seed 缺共同基础、表达监控或修复路线 | `repair` |
| `pain_regret_recovery_gate` | 痛苦/后悔 seed 缺恢复路线 | `repair` |
| `readiness_gate` | 出生准备度 feedback 直接晋升 | `critical_repair` |
| `rollback_quarantine_gate` | quarantine 被 release 但缺 release condition | `quarantine` |
| `anti_forgetting_gate` | 缺旧能力、旧关系、旧语言或旧 carrier feedback | `critical_repair` |
| `life_target_gate` | 任一真实生命目标缺 feedback | `critical_repair` |
| `handoff_gate` | 缺 second replay validation handoff | `hold_for_evidence` |

stage 只允许 `seed_ready`、`hold_for_evidence`、`repair`、`critical_repair`、`quarantine`。`seed_ready` 只代表可进入下一层验证，不代表出生完成。

## Acceptance Criteria

`01aq` 完成后，未来实现必须满足：

| 验收项 | 标准 |
|---|---|
| intake | `01ap` replay trace、shadow finding、future probe、blocked refs、stage refs 全部可读 |
| classification | 所有 replay/shadow 输出进入 12 类 feedback seed |
| truth gate | dream、inference、counterfactual、relationship、self rewrite、quarantine 材料有不同 route |
| language | 内言语、表达监控、共同语言、责任后悔语言进入 feedback seed |
| relationship | 关系 feedback 保留共同基础、承诺、信任修复和下一次对话 probe |
| dream/pain/regret | DreamFactGate、痛苦残留、后悔反事实、恢复路径闭合 |
| self/readiness | CoreContinuity、self rewrite proposal、readiness carrier gap 和 false open guard 闭合 |
| rollback/quarantine | release condition 与 rollback route 保留 |
| anti-forgetting | 旧能力、旧关系、旧语言、旧生命目标 carrier feedback 闭合 |
| life targets | 十项真实生命目标都有 next feedback seed |
| handoff | 输出 `prediction_active_inference_second_replay_validation` 和 `prediction_active_inference_next_cycle_patch_seed` |

## Downstream Handoff

本层向下游输出：

- `prediction_active_inference_second_replay_validation`：读取 feedback seed bundle、classification、arbitration policy、truth gate、DreamFactGate/CoreContinuity refs、anti-forgetting refs、rollback/quarantine refs、dashboard/stage refs，攻击 feedback seed 是否真实可承载。
- `prediction_active_inference_next_cycle_patch_seed`：读取 second replay validation 后通过的 observation/repair/growth/offline routes，生成下一轮主动预测 patch seed、runtime observation seed 和有限激活前置条件。

`01aq` 的结束状态不是终局，而是让 archive replay/shadow 结果变成下一轮生命反馈的入口。下一层必须继续检查：feedback seed 是否真的可执行，shadow finding 是否仍被封印，旧关系/旧语言/旧生命目标 carrier 是否被保护，梦境事实门和核心连续性是否阻断了错误晋升。
